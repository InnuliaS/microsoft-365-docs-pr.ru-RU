# <a name="plan-for-multi-factor-authentication-for-office-365-deployments"></a>Планирование многофакторной проверки подлинности для развертываний Office 365

Многофакторная проверка подлинности (MFA)  это проверка подлинности, выполняемая в несколько этапов и позволяющая реализовать дополнительный уровень защиты при входе пользователей и различных транзакциях. Она работает, требуя дополнительной проверочной проверки со сведениями, превышающими пароль для учетной записи пользователя, например:
  
- код доступа, сформированный случайным образом;
    
- телефонный звонок;
    
- смарт-карта (виртуальная или физическая); 
    
- биометрическое устройство. 
    
## <a name="mfa-in-office-365"></a>MFA в Office 365

Office 365 использует MFA для дополнительной безопасности и может управляться для каждой отдельной учетной записи пользователя в центре администрирования Microsoft 365. Office 365 предлагает следующее подмножество возможностей многофакторной проверки подлинности Azure в рамках подписки: 
  
- Возможность включения и применения MFA для конечных пользователей
    
- Использование мобильного приложения в качестве второго условия проверки подлинности (в Интернете и с помощью одноразового пароля).
    
- Телефонный звонок в качестве второго условия проверки подлинности.
    
- Использование SMS-сообщения в качестве второго условия проверки подлинности.
    
- Пароли приложений для клиентов, отличных от браузера (например, программное обеспечение Microsoft Lync 2013 Communications)
    
- Стандартное приветствие Майкрософт при проверке подлинности по телефону.
    
Полный список добавленных возможностей см. в статье [Принципы работы службы Azure Multi-Factor Authentication](https://go.microsoft.com/fwlink/?LinkId=506927) (раздел "Сравнение функций в разных версиях"). Для реализации полноценной многофакторной проверки подлинности вы можете приобрести службу Azure Multi-Factor Authentication. 
  
Вы получаете другое подмножество возможностей в зависимости от того, имеет ли облачное или гибридное удостоверение для Office 365 или федеративной проверки подлинности со службами федерации Active Directory (AD FS). 
  
|**Где можно управлять вашим клиентом Office 365?** | **Параметры второго фактора MFA**|

|:-----|:-----| | Только облако  <br/> | Многофакторная проверка подлинности Azure (текстовый или телефонный звонок)  <br/> | | Гибридная Настройка, управляемая локальная среда  <br/> | Если вы управляете удостоверением локального пользователя, возможны следующие варианты:  <br/>  Физическая или виртуальная смарт-карта (при использовании AD FS)  <br/> [Многофакторная проверка подлинности Azure](https://go.microsoft.com/fwlink/p/?LinkId=526677) (модуль для AD FS)  <br/>  Многофакторная проверка подлинности Azure Active Directory (Azure AD)  <br/> |
   
  
На рисунке ниже показано, как многофакторная проверка подлинности реализована в обновленных приложениях Office 2013 для устройств (на базе Windows). 

![Modern authentication for Office 2013 device apps.](../../media/dc37645c-b899-4715-b162-d7653bd0aebd.png)

Приложения для устройств Office 2013 поддерживают многофакторную проверку подлинности с помощью [библиотеки проверки подлинности Active Directory (ADAL)](https://go.microsoft.com/fwlink/p/?LinkId=526684). В Azure AD есть веб-страница, на которой пользователи могут выполнить вход. Удостоверения могут предоставляться Azure AD или федеративным поставщиком удостоверений, например AD FS. Для федеративных пользователей проверка подлинности выполняется описанным ниже образом.
  
1. Azure AD перенаправляет пользователя на веб-страницу входа, размещенную поставщиком удостоверений для записи клиента Office 365. Поставщик удостоверений определяется доменом, указанным в учетном имени пользователя.
    
2. Пользователь входит в систему на странице входа со своего устройства. 
    
3. Когда пользователь выполняет успешный вход, поставщик удостоверений возвращает маркер в Azure AD.
    
4. Azure AD возвращает приложению Office для устройств веб-маркер JSON (JWT), и приложение проходит проверку подлинности, используя этот веб-маркер в Office 365. 
    
  
## <a name="requirements-for-office-2013-client-apps"></a>Требования для клиентских приложений Office 2013

Чтобы включить многофакторную проверку подлинности для клиентских приложений Office 2013, должны быть установлены указанные ниже (или более поздние) версии программ. Требования к версиям зависят от того, как устанавливался Office: с помощью технологии [Версии "нажми и работай"](#click-to-run-based-installations) или же с помощью [Версии MSI](#msi-based-installations).
  
Чтобы определить, как был установлен Office:
  
1. Запустите Outlook 2013.
    
2. В меню **файл** выберите пункт **учетная запись Office**.
    
3. Если установка Outlook 2013 выполнялась по технологии "нажми и работай", вы увидите кнопку **Параметры обновления**. Если установка выполнялась с помощью MSI, кнопка **Параметры обновления** не отображается. 
    
    ![Как определить, является ли установка Office 2013 "нажми и работай" или на основе MSI](../../media/1e75143f-9e37-4e0c-9610-43a80771571e.png)

СОР дополнительные сведения см. в [статье вопросы и ответы о вики-статье по современной проверке подлинности](https://go.microsoft.com/fwlink/p/?LinkId=530064).

## <a name="enable-mfa"></a>Включение многофакторной проверки подлинности

Чтобы включить MFA для подписки на Office 365, выполните указанные ниже действия.
  
1. При необходимости [включите современная проверка подлинности для Office 2013 на устройствах с Windows](enable-modern-authentication.md).
    
2. Для федеративной проверки подлинности настройте многофакторную проверку подлинности Azure со сторонней службой каталогов.
    
    В [расширенных сценариях с многофакторной проверкой подлинности Azure и сторонними решениями VPN](https://docs.microsoft.com/azure/active-directory/authentication/howto-mfaserver-nps-vpn) для получения сведений об определенных поставщиках удостоверений, принятых этой программой. 
    
3. [Настройка многофакторной проверки подлинности для Office 365](set-up-multi-factor-authentication.md).
    
4. Сообщите пользователям, как [настроить MFA для своей учетной записи пользователя Office 365](https://support.office.com/article/set-up-2-step-verification-for-office-365-ace1d096-61e5-449b-a875-58eb3d74de14). После настройки вторичного метода проверки подлинности для их будущих входов потребуется MFA.
    
> [!IMPORTANT]
> Если вы включили пользователей для многофакторной проверки подлинности Azure и у них есть устройства, на которых работает Office 2013, для которых не включена современная проверка подлинности, им потребуется использовать пароли приложений. Дополнительные сведения о паролях приложений, а также о том, где/где и как их использовать, можно найти здесь: " [пароли приложений с помощью Azure Multi_Factor проверки подлинности](https://go.microsoft.com/fwlink/p/?LinkId=528178)".
