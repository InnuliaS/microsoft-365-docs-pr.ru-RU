# <a name="plan-for-multi-factor-authentication-for-microsoft-365-deployments"></a>Планирование многофакторной проверки подлинности для развертываний Microsoft 365

Многофакторная проверка подлинности (MFA)  это проверка подлинности, выполняемая в несколько этапов и позволяющая реализовать дополнительный уровень защиты при входе пользователей и различных транзакциях. Она работает, требуя дополнительной проверочной проверки со сведениями, превышающими пароль для учетной записи пользователя, например:
  
- код доступа, сформированный случайным образом;
    
- телефонный звонок;
    
- смарт-карта (виртуальная или физическая); 
    
- биометрическое устройство. 
    
## <a name="multi-factor-authentication-in-microsoft-365"></a>Многофакторная проверка подлинности в Microsoft 365

Microsoft 365 использует многофакторную проверку подлинности для обеспечения дополнительной защиты и управления ими из центра администрирования Microsoft 365. Microsoft 365 предлагает следующее подмножество возможностей многофакторной проверки подлинности Azure в рамках подписки: 
  
- Возможность включения и применения MFA для конечных пользователей
    
- Использование мобильного приложения в качестве второго условия проверки подлинности (в Интернете и с помощью одноразового пароля).
    
- Телефонный звонок в качестве второго условия проверки подлинности.
    
- Использование SMS-сообщения в качестве второго условия проверки подлинности.
    
- Пароли приложений для клиентов, отличных от браузера (например, программное обеспечение Microsoft Lync 2013 Communications)
    
- Стандартное приветствие Майкрософт при проверке подлинности по телефону.
    
Полный список добавленных возможностей см. в статье [Принципы работы службы Azure Multi-Factor Authentication](https://go.microsoft.com/fwlink/?LinkId=506927) (раздел "Сравнение функций в разных версиях"). Для реализации полноценной многофакторной проверки подлинности вы можете приобрести службу Azure Multi-Factor Authentication. 
  
В зависимости от наличия облачного развертывания для Microsoft 365 или гибридной среды с единым входом и службами федерации Active Directory (AD FS) вы получаете другое подмножество возможностей. 
  
|**Где вы управляете Microsoft 365?**|**Варианты второго условия многофакторной проверки подлинности**|
|:-----|:-----|
|Только в облаке  <br/> |Azure Active Directory MFA (текстовое сообщение или телефонный звонок)  <br/> |
|Гибридная среда, управляемая локально  <br/> | Если вы управляете удостоверениями пользователей локально, вам могут быть доступны следующие варианты:  <br/>  физическая или виртуальная смарт-карта (AD FS);  <br/> [Azure MFA](https://go.microsoft.com/fwlink/p/?LinkId=526677) (модуль для AD FS)  <br/>  Azure AD MFA  <br/> |
   
  
На рисунке ниже показано, как многофакторная проверка подлинности реализована в обновленных приложениях Office 2013 для устройств (на базе Windows). Все приложения Office 2013 для устройств поддерживают многофакторную проверку подлинности с помощью [библиотеки проверки подлинности Active Directory (ADAL)](https://go.microsoft.com/fwlink/p/?LinkId=526684). В Azure AD есть веб-страница, на которой пользователи могут выполнить вход. Удостоверения могут предоставляться Azure AD или федеративным поставщиком удостоверений, например AD FS. Для федеративных пользователей проверка подлинности выполняется описанным ниже образом.
  
1. Azure AD перенаправляет пользователя на веб-страницу входа, размещенную поставщиком удостоверений записи для Организации. Поставщик удостоверений определяется доменом, указанным в учетном имени пользователя.
    
2. Пользователь входит в систему на странице входа со своего устройства. 
    
3. Когда пользователь выполняет успешный вход, поставщик удостоверений возвращает маркер в Azure AD.
    
4. Azure AD возвращает веб-маркер JSON (JWT) приложению Office Device, а проверка подлинности приложения для устройства осуществляется с помощью JWT с Microsoft 365. 
    
  
## <a name="requirements-for-office-2013-client-apps"></a>Требования для клиентских приложений Office 2013

Чтобы включить многофакторную проверку подлинности для клиентских приложений Office 2013, должны быть установлены указанные ниже (или более поздние) версии программ. Требования к версиям зависят от того, как устанавливался Office: с помощью технологии [Версии "нажми и работай"](#click-to-run-based-installations) или же с помощью [Версии MSI](#msi-based-installations).
  
Чтобы определить, как был установлен Office:
  
1. Запустите Outlook 2013.
    
2. В меню **файл** выберите пункт **учетная запись Office**.
    
3. Если установка Outlook 2013 выполнялась по технологии "нажми и работай", вы увидите кнопку **Параметры обновления**. Если установка выполнялась с помощью MSI, кнопка **Параметры обновления** не отображается.

СОР дополнительные сведения см. в [статье вопросы и ответы о вики-статье по современной проверке подлинности](https://go.microsoft.com/fwlink/p/?LinkId=530064).

## <a name="enable-mfa"></a>Включение многофакторной проверки подлинности

Чтобы включить MFA для подписки, выполните указанные ниже действия.
  
1. При необходимости [включите современная проверка подлинности для Office 2013 на устройствах с Windows](enable-modern-authentication.md).
    
  - Настройте Azure MFA с помощью сторонних служб каталогов.
    
    В [расширенных сценариях с многофакторной проверкой подлинности Azure и сторонними решениями VPN](https://docs.microsoft.com/azure/active-directory/authentication/howto-mfaserver-nps-vpn) для получения сведений об определенных поставщиках удостоверений, принятых этой программой. 
    
2. [Настройка многофакторной проверки подлинности для Microsoft 365](set-up-multi-factor-authentication.md)
    
3. Расскажите отдельным пользователям, как войти в MFA: [Войдите в Microsoft 365 с 2 этапной проверкой](https://support.office.com/article/2b856342-170a-438e-9a4f-3c092394d3cb.aspx).

> [!IMPORTANT]
> Если вы включили для своих пользователей службу Azure AD MFA и они работают в Office 2013 на каких-либо устройствах, для которых не предусмотрена современная проверка подлинности, им потребуются пароли приложений. Дополнительные сведения о паролях приложений и их использовании см. в статье [Пароли приложения в Azure Multi-Factor Authentication](https://go.microsoft.com/fwlink/p/?LinkId=528178).
