## <a name="android-and-ios-application-protection-settings"></a>Параметры защиты приложений для Android и iOS

 **Важные:** Подписки на Microsoft 365 Business предоставляет лицензии для изменения параметров Intune. Просмотреть [Общие сведения о Intune для начала.](https://docs.microsoft.com/intune/introduction-intune)

## <a name="windows-10-app-protection-settings"></a>Параметры защиты приложений для Windows 10

В приведенной ниже таблице показано сопоставление параметров политики приложений для Windows 10 с параметрами Intune.
  
Intune параметр при входе, используя свои учетные данные администратора Microsoft 365 для бизнеса в разделе [Azure портала](https://portal.azure.com), а затем выберите **несколько служб**и тип в Intune в **Фильтр** **Защиты приложения Intune** \> ** Политики приложения**.
  
 **Важно!** Подписка на Microsoft 365 бизнес включает лицензию на изменение только тех параметров Intune, которые соответствуют параметрам, доступным в Microsoft 365 бизнес. 
  
Щелкните имя нужной политики и выберите пункт **Общие, Назначения**, **Разрешенные приложения**, **Исключенные приложения**, **Обязательные параметры** или **Дополнительные параметры** на левой панели навигации, чтобы просмотреть доступные параметры. 
  
|**Параметр политики приложений для Windows 10**|**Параметры Intune**|
|:-----|:-----|
|Шифровать рабочие файлы  <br/> |**Дополнительные параметры** \> **Защита данных**: для параметров **Отзывать ключи шифрования при отмене регистрации** и **Отзывать доступ к защищенным данным при регистрации устройства в MDM** устанавливается значение **Вкл.**  <br/> |
|Запретить пользователям копирование данных компании на личные файлы.  <br/> |**Обязательные параметры** \> **Режим защиты Windows Information Protection**: значение **Вкл.** в Office 365 бизнес соответствует значению **Скрыть переопределения**, а значение **Откл.** в Office 365 бизнес соответствует значению **Откл.**<br/> |
|Управление доступом к документам Office  <br/> | Если в Office 365 бизнес этот параметр имеет значение **Вкл.**, тогда для параметра  <br/> **Использовать Windows Hello для бизнеса для входа в Windows** в разделе \> **** **Доступ** устанавливается значение **Вкл.**. Кроме того, задаются следующие дополнительные параметры:    <br/> **Задать минимальное число символов для ПИН-кода**: **4**  <br/> **Настроить использование прописных букв в ПИН-коде Windows Hello для бизнеса**: **Запретить использование прописных букв в ПИН-коде**  <br/> **Настроить использование строчных букв в ПИН-коде Windows Hello для бизнеса**: **Запретить использование строчных букв в ПИН-коде**  <br/> **Настроить использование специальных символов в ПИН-коде Windows Hello для бизнеса**: **Запретить использование специальных символов в ПИН-коде**  <br/> **Укажите интервал времени (в днях), в течение которого можно использовать ПИН-код до ого, как система потребует его изменить**: **0**  <br/> **Укажите число запрещенных к использованию предыдущих значений ПИН-кода, связываемых с учетной записью пользователя**: **0**  <br/> **Число неудачных попыток проверки подлинности до очистки устройства**: то же значение, что и в Office 365 бизнес (по умолчанию  5)  <br/> **Максимальное количество времени (в минутах) пр простое до блокировки устройства с помощью ПИН-кода или пароля**: то же значение, что и в Office 365 бизнес  <br/> |
