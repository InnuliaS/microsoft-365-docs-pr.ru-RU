# <a name="self-service-purchase-faq"></a>Вопросы и ответы по самостоятельной покупке

> [!NOTE]
> Сведения, приведенные в этой статье, применимы только к подпискам на Microsoft Power Platform (Power BI, PowerApps и Flow).

## <a name="general"></a>Общие

### <a name="what-changes-did-microsoft-announce-around-self-service-purchases-for-the-power-platform-products"></a>Какие изменения были объявлены корпорацией Майкрософт для самостоятельного приобретения продуктов для платформы Power Platform?

Начиная с 19 ноября, 2019, возможности самостоятельного приобретения, подписки и управления лицензиями для продуктов Power Platform (Power BI, PowerApps и Flow) будут доступны большинству коммерческих облачных клиентов в США. Самостоятельная Покупка дает пользователям возможность испытать новые технологии и позволять им разрабатывать решения, которые в конечном итоге повышают эффективность работы с крупными организациями. В настоящее время эта возможность будет недоступна для клиентов, которые не являются правительственными, невыгодными или образованными. Центральные отделы снабжения и ИТ будут иметь видимость для всех пользователей, которые покупают и развертывают самостоятельные решения для приобретения через <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центр администрирования Microsoft 365</a>.

### <a name="why-is-microsoft-adding-a-self-service-purchase-option-for-the-power-platform-products"></a>Почему корпорация Майкрософт добавляет функцию приобретения самообслуживания для продуктов Power Platform?

В результате цифрового преобразования традиционные роли меняются. Так как сотрудники становятся более независимыми и более совершенными в технологиях, мы видели повышенную потребность от пользователей и организаций, чтобы позволить пользователям покупать подписки самостоятельно. Цель приобретения самообслуживания состоит в том, чтобы позволить пользователям разрабатывать собственные решения для разблокировки продуктивности и влияния на бизнес, а также для обеспечения соответствия требованиям организации и управлению данными.

### <a name="when-will-self-service-purchase-for-the-power-platform-products-be-available"></a>Когда будет доступна самостоятельная покупка продуктов для платформы Power Platform?

Корпорация Майкрософт запускает самостоятельную покупку для Power BI на 19 ноября 2019 до клиентов в США, благодаря которым дополнительные рынки становятся доступны в ближайшие месяцы. PowerApps и Microsoft Flow будут добавлены 4 декабря 2019 г. В настоящее время эта возможность будет недоступна для клиентов, которые не являются правительственными, невыгодными или образованными.

### <a name="will-self-service-purchase-be-enabled-for-services-beyond-the-power-platform-products"></a>Будет ли осуществляться самостоятельная покупка для служб за пределами продуктов Power Platform?

В настоящее время семейство продуктов Power Platform является единственной службой, предлагаемой через самостоятельную покупку.

## <a name="making-a-self-service-purchase"></a>Самостоятельная Покупка

### <a name="how-does-a-customer-make-a-self-service-purchase"></a>Как клиент делает самостоятельную покупку?

Пользователи смогут выполнять самостоятельную покупку через Интернет на веб-сайтах Power BI, PowerApps и Flow. Пользователям сначала потребуется ввести адрес электронной почты, чтобы убедиться в том, что они являются пользователями существующего клиента Azure Active Directory (AD), а затем они будут направлены на вход, используя свои учетные данные Azure AD. После входа в систему клиенту будет предложено выбрать количество подписок, которые они хотят приобрести и предоставить платеж по кредитной карте. По завершении покупки он сможет начать использовать свою подписку. Покупатель также сможет получить доступ к ограниченному представлению <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центра администрирования Microsoft 365</a> , где они могут разрешить другим людям в Организации использовать продукт.

### <a name="what-are-the-payment-options-for-self-service-purchases"></a>Каковы варианты оплаты при самостоятельных покупках?

В настоящее время кредитная карта — единственный доступный способ оплаты. Оплата через выставление счетов не поддерживается.

### <a name="who-can-buy-through-self-service-purchase"></a>Кто может покупать через самостоятельную покупку?

Любой пользователь с учетной записью пользователя, не являющегося гостевым, в управляемом клиенте Azure AD можно купить. В настоящее время эта возможность будет недоступна для клиентов, которые не являются правительственными, невыгодными или образованными. Пользователи в организациях или рынках, у которых нет прав на самостоятельную закупку, будут видеть сообщение, предлагающее им связаться с ИТ ИТ, как сегодня.

### <a name="can-guest-users-buy-through-self-service-purchase"></a>Могут гостевые пользователи покупать самостоятельную закупку?

Нет, гостевые пользователи не могут выполнять самостоятельную покупку в клиенте, в котором они являются гостями.

### <a name="can-users-synced-from-an-on-premises-active-directory-buy-through-self-service-purchase"></a>Можно синхронизировать пользователей из локальной службы Active Directory с помощью самостоятельного приобретения?

Если у пользователя есть активная учетная запись пользователя в подходящего клиента Azure AD, она может выполнить самостоятельную покупку.

### <a name="who-can-self-service-purchasers-assign-licenses-to"></a>Кому могут самостоятельно предоставлять покупатели лицензии на самостоятельное обслуживание?

Самостоятельные покупатели смогут назначать лицензии пользователям только в том же клиенте Azure AD. Покупатель сможет получить доступ к ограниченному представлению <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центра администрирования Microsoft 365</a> для назначения лицензий. Они имеют только видимость и могут назначать лицензии тем продуктам, которые они приобрели через самостоятельную покупку, и могут назначать эти лицензии пользователям в том же клиенте Azure AD.

### <a name="where-does-the-self-service-purchaser-see-and-manage-their-purchases"></a>Где самостоятельный покупатель видит и управляет покупками?

Самостоятельные покупатели могут управлять покупками в ограниченном представлении <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центра администрирования Microsoft 365</a>. Покупатели всегда могут получить доступ к центру администрирования из плитки **администратора** в средстве запуска приложений Office 365, встроенных во все приложения Office 365 и Dynamics Online. Они могут просматривать сделанные покупки, покупать дополнительные подписки на одну и ту же службу и назначать лицензии для этих подписок другим пользователям в Организации. Кроме того, покупатели могут просмотреть и оплатить свой счет, обновить способ оплаты и отказаться от их подписки.

## <a name="pricing"></a>Цены

### <a name="what-is-the-pricing-for-self-service-purchases"></a>Каковы цены для самостоятельных покупок?

Цены для каждой из продуктов Power Platform для самостоятельных покупок будут доступны на веб-сайте корпорации Майкрософт, и они также отображаются в процессе извлечения при совершении покупки самостоятельно. Эти цены могут отличаться от цен, которые Организация оплачивает при создании центральных покупок или цен, предлагаемых через партнера.

### <a name="who-is-responsible-for-payment"></a>Кто отвечает за оплату?

Сотрудник, который покупает подписку через самостоятельную покупку, будет оплачиваться и несет ответственность за оплату на основе терминов и ценообразования покупки.

## <a name="admin-capabilities"></a>Возможности администрирования

### <a name="what-capabilities-does-an-admin-have-for-self-service-purchases"></a>Какие возможности имеет администратор для самостоятельного приобретения?

Администраторы могут просматривать все самостоятельные покупки, сделанные в Организации, в <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центре администрирования Microsoft 365</a>. Они могут видеть название продукта, покупателя, подписки на покупку, дату истечения срока действия, историю заказов, покупную цену и пользователей, назначенных для каждой самостоятельной покупки.

Несмотря на то, что все пользователи Azure AD в соответствующих организациях могут выполнять самостоятельную покупку и иметь полный контроль над подписками, администраторы имеют одинаковые политики управления данными и доступа к продуктам, приобретенным через самостоятельную покупку или централизованно.

### <a name="why-are-self-service-purchases-not-subject-to-admin-approval"></a>Почему самостоятельное приобретение не подвергается одобрению администратора?

Мы реагируем на наших клиентов, которые запросили эту возможность, и позволяют администраторам управлять службами и учитывать управление данными и соответствием требованиям. Чтобы узнать больше об управлении субъектами службы Azure AD, ознакомьтесь с [РазMsolServicePrincipal Set –](https://docs.microsoft.com/powershell/module/msonline/set-msolserviceprincipal?view=azureadps-1.0). Кроме того, администраторы имеют полное представление о том, кто выполнил самостоятельную покупку и какие пользователи были назначены лицензии на самостоятельную покупку. Организации могут использовать собственные внутренние политики, процедуры и коммуникации, чтобы убедиться, что сотрудники, обеспечивающие самостоятельную покупку, соответствуют политикам компании. 

### <a name="how-is-microsoft-respecting-data-governance-and-compliance-by-enabling-self-service-purchase"></a>Как корпорация Майкрософт уважает управление данными и обеспечивает соответствие требованиям, разрешая самостоятельную покупку?

Администраторы поддерживают контроль над тем, какие службы и продукты включены в своем клиенте на основе требований к управлению данными и соответствию требованиям. Кроме того, все политики управления данными и доступа, которые ваша организация включили, продолжат применять к самостоятельно приобретенным службам с поддержкой.

### <a name="who-owns-the-product-data-created-from-self-service-purchases"></a>Кто владеет данными продукции, созданными с помощью самостоятельных покупок?

Данные, созданные на основе продуктов, приобретенных через самостоятельную закупку, принадлежать Организации и управляются им.

### <a name="how-do-i-centralize-the-purchases-made-through-self-service-purchase"></a>Как централизовать покупки, совершенные с помощью самостоятельной покупки?

Администраторы могут назначать существующие лицензии или покупать дополнительные подписки на продукты Power Platform (Power BI, PowerApps и Microsoft Flow) через существующие соглашения и цены для пользователей, которым назначена самостоятельная покупка. После назначения этих централизованно приобретенных лицензий администраторы могут запросить, чтобы покупатели отменили существующие подписки. Корпорация Майкрософт исследует способы упрощения и упрощения этого процесса для администраторов в будущем.

### <a name="where-does-the-admin-see-self-service-purchases"></a>Где администратор видит самостоятельные приобретения услуг?

Глобальные администраторы и администраторы выставления счетов могут просматривать подписки, приобретаемые с помощью самостоятельной покупки в продуктах для **выставления счетов** > **& службы** в <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центре администрирования Microsoft 365</a> , а также все остальные подписки, приобретенные через центр приобретения. Они могут отфильтровать список только по подпискам, приобретенным через центральную закупку, или включить подписки, приобретаемые с помощью самостоятельной покупки.

Администраторы могут видеть продукт, название покупателя, приобретенную подписку, дату истечения срока действия, историю заказов, покупную цену и назначенные пользователи.

## <a name="support-and-training"></a>Поддержка и обучение

### <a name="are-customers-it-departments-or-partners-expected-to-support-products-bought-through-self-service-purchase"></a>Являются ли ИТ отделы или партнеры, которые должны поддерживать продукты, приобретаемые через самостоятельную покупку услуг?

ИТ отделы и партнеры не должны предоставлять поддержку продуктов, приобретенных с помощью самостоятельной покупки. Корпорация Майкрософт предоставит стандартную поддержку покупателей самостоятельных служб.

### <a name="how-are-users-expected-to-receive-training-on-the-products-they-buy-through-self-service-purchase"></a>Как пользователи должны будут получать обучающие материалы по продуктам, которые они приобрели через самостоятельную покупку?

Подробное обучение пользователей предоставляется на веб-сайтах Microsoft Power BI, PowerApps и Flow. У продуктов есть обучающие материалы, документация, примеры и надежные сообщества для получения ответов и советов непосредственно от других пользователей.

### <a name="what-happens-to-a-self-service-purchase-if-a-user-leaves-the-organization"></a>Что происходит с самостоятельным приобретением, если пользователь покидает организацию?

Действительные пользователи продолжат все время приобретения самообслуживания на время подписки. Подписка остается активной до тех пор, пока покупатель не отменяет ее или не отменяет запрос администратора о том, что подписка будет отменена через службу поддержки клиентов. Кроме того, администраторы могут назначить централизованно приобретенную лицензию пользователям с отмененной подпиской.

## <a name="partners"></a>Партнеры

### <a name="whats-the-role-of-microsofts-partners-in-self-service-purchases"></a>Какова роль партнеров корпорации Майкрософт в самостоятельных покупках?

Партнеры, у которых есть делегированные права на администрирование, могут видеть самостоятельные приобретения в <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центре администрирования Microsoft 365</a>, как и администратор. Партнеры могут поддерживать организацию, которая хочет централизовать продукты, приобретаемые с помощью самостоятельных покупок. Кроме того, партнеры могут предоставить решения для расширения возможностей самостоятельных покупок.
