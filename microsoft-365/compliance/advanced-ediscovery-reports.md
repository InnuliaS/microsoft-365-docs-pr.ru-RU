# <a name="advanced-ediscovery-reports-preview"></a>Расширенные отчеты об обнаружении электронных данных (Предварительная версия)

Расширенные отчеты обнаружения электронных данных помогают объединить данные вариантов в Организации для упрощения анализа данных и отчетности Организации. Функция Advanced eDiscovery Reports содержит несколько встроенных отчетов, которые помогут ответить на такие вопросы:

- Сколько активных custodians есть для всех случаев в моей организации?

- Сколько источников данных находится на удержании для всех случаев в Организации?

- Сколько уведомлений о удержании было выдано в прошлом месяце для всех случаев в Организации?

- Сколько активных и закрытых обращений в моей организации?

## <a name="before-you-begin"></a>Перед началом работы

- Для доступа к расширенным отчетам обнаружения электронных данных необходимо быть участником группы ролей "Администратор обнаружения электронных данных". Участник этой группы ролей предоставляет необходимые разрешения для просмотра, фильтрации и экспорта данных в отчетах. Дополнительные сведения см. в статье [Назначение разрешений на обнаружение электронных данных](assign-ediscovery-permissions.md).

- Расширенные отчеты обнаружения электронных данных обновляются ежедневно. В результате может возникнуть задержка при создании новых случаев, custodians, источников данных и коммуникаций, а также при их появления в соответствующем отчете.

Чтобы получить доступ к расширенным отчетам обнаружения электронных данных:

1. Перейдите по ссылке https://protection.office.com.
  
2. Войдите с помощью рабочей или учебной учетной записи.
  
3. В центре безопасности & соответствия требованиям выберите **Обнаружение электронных данных > Advanced eDiscovery**.
  
   На домашней странице " **Расширенное обнаружение электронных** данных" вкладки "обращение", "Хранитель", "источник данных" и "отчет о связи" отображаются в верхней части страницы. 
  
   ![Расширенные отчеты обнаружения электронных данных на домашней странице](../media/report-home.png)

5. Чтобы просмотреть отчет, перейдите на вкладку отчет и при необходимости можно выполнить одно из следующих действий:

   ![Вы можете фильтровать или загружать данные отчета](../media/AeDReportsFilterDownload.png)

   а) Нажмите кнопку **Фильтр** для сужения данных отчета. Для каждого отчета можно выполнить фильтрацию по разным свойствам.
  
   б) Нажмите **скачать в CSV** , чтобы экспортировать данные отчета в CSV-файл.

## <a name="case-report"></a>Отчет о делах

В отчете о делах объединяются сведения об активных и закрытых случаях обнаружения электронных данных в Организации.

|Столбец        |Описание|
|:-------------|:-------------|
|Идентификатор обращения | Уникальный идентификатор каждого случая.| 
|Имя дела | Определенное пользователем имя обращения.|
|Состояние | Указывает, является ли обращение активным или закрытым.|
|Дата создания |Дата создания обращения. Даты представлены в формате UTC.|
|Дата последнего изменения |Дата, когда обращение было закрыто или Последнее обновление. Даты представлены в формате UTC.|

## <a name="custodian-report"></a>Отчет хранитель

В рабочем процессе обнаружения электронных данных лица, являющиеся субъектами юридических дел или расследования, называются *Data custodians* (или просто *custodians*) и определяются как "лица, обладающие административным управлением документа или электронного файла". С помощью отчета хранитель можно определить все custodians, чьи источники данных помещаются в удержание для всех случаев в Организации.

|Столбец         |Описание|
|:-------------|:-------------|
|Имя хранитель| Имя хранитель в Active Directory.|
|Имя участника-пользователя хранитель | Имя участника пользователя хранитель.|
|Идентификатор хранитель | Уникальный идентификатор хранитель в заданном случае. |
|Имя дела | Определенное пользователем имя варианта.|
|Состояние удержания | Указывает, находится ли хранитель в настоящее время на удержании или в случае, если они были выпущены из этого случая.|
|Идентификатор обращения | Уникальный идентификатор для случая.|
|Состояние связи |Указывает, было ли для хранитель выдано уведомление о юридическом удержании. |
|Добавлена хранитель | Дата добавления хранитель в обращение. Даты представлены в формате UTC.|
|||

## <a name="data-source-report"></a>Отчет источника данных

Вы можете использовать расширенное дело eDiscovery для создания удержаний, чтобы сохранить содержимое, которое может быть релевантно для случая. С помощью расширенных возможностей удержания обнаружения электронных данных можно размещать удержания в custodians и их источниках данных. Кроме того, вы можете поместить в почтовые ящики и учетные записи OneDrive для бизнеса, не кустодиал удержание. Вы можете использовать отчет источники данных для объединения сведений о расположении контента при удержании для всех случаев в Организации.

|Столбец        |Описание|
| -------------|-------------|
|Идентификатор обращения |Уникальный идентификатор каждого случая. |
|Workload |Указывает тип расположения содержимого, размещенного на удержании (например, Exchange или SharePoint).
|Имя расположения |Указывает SMTP-адрес (для почтовых ящиков Exchange) или URL-адрес (для сайтов SharePoint) расположения контента. | 
|Идентификатор хранитель |Если источник данных принадлежит хранитель, в этом столбце отображается уникальный идентификатор хранитель в заданном случае. Этот столбец будет иметь значение NULL для расположений, не являющихся кустодиал.|
|Имя хранитель |Имя хранитель в Active Directory.| 
|Имя дела |Определенное пользователем имя варианта.| 
|Состояние |Указывает, находится ли расположение содержимого в данный момент на удержании. | 
|Идентификатор политики удержания |Уникальный идентификатор удержания, который содержит расположение контента. | 
|Дата создания удержания |Указывает дату создания политики удержания. Даты представлены в формате UTC. | 
|Имя политики хранения |Имя удержания, которое содержит расположение контента. |
|Дата изменения удержания |Дата последнего изменения удержания. Даты представлены в формате UTC.| 
|Сохранить последние изменения|Имя пользователя, который последним изменил удержание.| 
|||

## <a name="communication-report"></a>Отчет о связи

Ваша организация может выдать уведомления о юридических удержаниях, чтобы уведомить custodians о своих обязательствах, чтобы сохранить соответствующие сведения в рамках судебного разбирательства или расследования. Отчет о связи можно использовать для просмотра статистических данных на подтверждениях, напоминаниях, эскалациях и других типах общения.

|Столбец         |Описание|
| -------------|-------------|
|Идентификатор обращения | Уникальный идентификатор для случая.|
|Имя дела | Определенное пользователем имя обращения.|
|Идентификатор хранитель |Уникальный идентификатор для хранитель в заданном случае.|
|Имя хранитель |Имя хранитель.|
|Тип уведомления |Указывает тип уведомления, выданного хранитель.|
|Выдача директора |Имя пользователя, который выдал уведомление о юридическом удержании.|
|Событие уведомления|Указывает сообщение об официальном удержании, отправленное пользователю. Возможные значения: напоминание, эскалация, подтверждение и выдача удержаний.|
|Дата отправки |Дата выдачи связи. Для подтверждений этот столбец указывает время, в течение которого уведомление было подтверждено хранитель. Даты представлены в формате UTC.|
|||
