# <a name="customize-compliance-score-preview-with-assessments"></a>Настройка оценки соответствия требованиям (Предварительная версия) с оценками

**В этой статье:** Узнайте, как настроить оценку соответствия требованиям, настроив готовые к использованию **оценки**. Узнайте, как изменить **шаблон** для оценки и создать собственные пользовательские оценки в соответствии с потребностями Организации. В этой статье также рассказывается, как упорядочивать оценки по **группам**, работать с **элементами управления**и экспортировать **отчеты**оценки.

## <a name="introduction-to-assessments"></a>Общие сведения о оценках

Оценка соответствия позволяет управлять соответствием нормативным требованиям и сертификациям, которые применяются к Организации. Оценки — это группировка элементов управления в определенном нормативном, стандартном или политике. Оценка соответствия упрощает отслеживание соответствия требованиям, предоставляя готовые к использованию оценки, охватывающие различные отраслевые и региональные нормативные нормы и сертификаты.

Каждая оценка создается на основе шаблона, который служит в качестве платформы, содержащей необходимые элементы управления и действия по улучшению для выполнения оценки. Настройка наиболее релевантных оценок для Организации поможет убедиться в том, что вы реализуете политики и рабочие процедуры, которые могут ограничивать риск для обеспечения соответствия требованиям.

Все ваши оценки перечислены на странице "оценки". [Узнайте больше](compliance-score-setup.md#assessments-page) о том, как фильтровать представления оценки и состояния интерпретации.

## <a name="data-protection-baseline-default-assessment"></a>Базовая оценка защиты данных по умолчанию

Чтобы приступить к работе, корпорация Майкрософт предоставляет оценку **по умолчанию** в показателе соответствия требованиям, который содержит базовый план защиты данных Microsoft 365. Этот базовый план представляет собой набор элементов управления, включающих ключевые нормативные требования и стандарты для защиты данных и общего управления данными. Этот базовый план рисует элементы в основном с помощью NIST CSF (Национальный институт стандартов и технологий Циберсекурити Framework) и ISO (Международная организация по стандартизации), а также из FedRAMP (Федеральный показатель рисков и программного обеспечения управления авторизацией) и GDPR (общее количество требований к защите данных Европейского союза).

Эта оценка используется для вычисления начального показателя при первом достижении оценки соответствия требованиям перед настройкой других оценок. Оценка соответствия собирает начальные сигналы от решений Microsoft 365. Вы увидите, как выполняется Организация относительно основных стандартов защиты данных и нормативных требований, а также ознакомьтесь с предлагаемыми действиями по улучшению.

Так как каждая организация обладает особыми потребностями, для оценки соответствия требованиям необходимо настроить собственные оценки и управлять ими, чтобы свести к минимуму риски и уменьшить риски.

## <a name="assessment-creation-overview"></a>Обзор создания оценки

Вы можете настроить оценки тремя способами:

1. Выберите готов к использованию оценки.
2. Измените [шаблон оценки](compliance-score-templates.md) в соответствии с вашими потребностями.
3. Создание собственной настраиваемой оценки.

Для создания или изменения оценок пользователи должны иметь роль глобального администратора, администратора соответствия требованиям, администратора данных соответствия требованиям или администратора безопасности. Дополнительные сведения о [ролях и разрешениях](compliance-score-setup.md#set-user-permissions-and-assign-roles).

> [!NOTE]
> Все оценки, созданные или измененные в показателе соответствия требованиям, также будут отражены в диспетчере соответствия требованиям. Узнайте больше о [связи между показателем соответствия требованиям и диспетчером соответствия требованиям](compliance-score.md#relationship-to-compliance-manager).

### <a name="ready-to-use-assessments"></a>Готовность к использованию оценок

Начинать ваше путешествие по соответствию требованиям, выбрав [один из оценок](compliance-score-templates.md) соответствия требованиям, чтобы создать оценки. Шаблоны содержат элементы управления и действия по улучшению, необходимые для каждой конкретной оценки. Шаблоны рейтинга соответствия требованиям охватывают нормативы и сертификаты, которые соответствуют отраслям, регионам и распространенным стандартам защиты данных, таким как GDPR и ISO 27001.

**Чтобы настроить оценку**, выберите один из шаблонов при начале [построения оценки с помощью мастера](#create-an-assessment).

### <a name="modify-the-template-of-an-assessment"></a>Изменение шаблона оценки

Вы можете изменять шаблоны оценки соответствия требованиям для оценки, чтобы лучше соответствовать потребностям вашей организации. Например, если обычно требуется соответствие требованиям закона HIPAA, но требуется дополнительная защита данных или безопасность, вы можете использовать наш шаблон HIPAA и добавить собственные настраиваемые поля для создания новой оценки, которая следит за ходом выполнения. Во время общедоступной предварительной версии необходимо перейти к диспетчеру соответствия требованиям, чтобы изменить шаблоны.

**Чтобы изменить шаблон оценки**, выполните следующие действия:

1. Просмотрите список доступных [шаблонов](compliance-score-templates.md) в показателе соответствия требованиям, чтобы определить, какой из них вы хотите изменить.
2. [Чтобы настроить шаблон с помощью процесса расширения, ознакомьтесь с инструкциями по диспетчеру соответствия требованиям](working-with-compliance-manager.md#customize-a-template-through-the-extension-process).
3. После того как вы создали шаблон и импортировали его в диспетчере соответствия требованиям, вы можете создать новую оценку в рейтинге соответствия требованиям. Выполните процесс построения оценки с помощью [мастера создания оценки](#create-an-assessment).

> [!NOTE]
> Узнайте больше о [связи между рейтингом соответствия требованиям и диспетчером соответствия требованиям](compliance-score.md#relationship-to-compliance-manager) во время общедоступной предварительной версии.

### <a name="create-your-own-custom-assessment"></a>Создание собственной настраиваемой оценки

Вы можете самостоятельно самостоятельно создать свою оценку для точного отслеживания потребностей Организации. Как и в описанном выше процессе изменения, для создания собственной оценки необходимо сначала создать собственный шаблон для оценки в диспетчере соответствия требованиям.

**Чтобы создать пользовательскую оценку**, выполните следующие действия:

1. Узнайте, как [создать собственный шаблон в диспетчере соответствия требованиям](working-with-compliance-manager.md#create-your-own-template-and-import-it-into-compliance-manager).
2. После того как вы создали шаблон и импортировали его в диспетчере соответствия требованиям, вы можете создать новую оценку в рейтинге соответствия требованиям. Выполните процесс построения оценки с помощью [мастера создания оценки](#create-an-assessment).

## <a name="understand-groups-before-creating-assessments"></a>Общие сведения о группах перед созданием оценок

Перед созданием или изменением оценок важно понимать, как работают группы. При создании оценки необходимо назначить ее группе во время этого процесса. Поэтому перед созданием оценок рекомендуется спланировать стратегию группировки для оценки.

### <a name="what-are-groups"></a>Что такое группы

Группы — это контейнеры, позволяющие упорядочивать оценки. Вы можете группировать оценки логически для вас, например, по годам или нормативам, либо на основе разделений Организации или географических регионов. Ниже приведены примеры двух групп и их базовых оценок:

- **ФФИЕК — оценки 2020**
  - Office 365 + ФФИЕК —
  - Intune + ФФИЕК —
- **Оценка безопасности данных и конфиденциальности**
  - Office 365 + ISO 27001:2013
  - Office 365 + ISO 27018:2014

При двух различных оценках, выполняемых вами в одной группе, любые обновления, вносимые в сведения о реализации действия или состояние, будут автоматически синхронизированы с одним действием в любой другой оценке в группе. Эта синхронизация позволяет реализовать одно действие по улучшению и отвечать нескольким требованиям в отношении нескольких нормативных требований.

### <a name="how-to-create-a-group"></a>Создание группы

В процессе [создания новой оценки](#create-an-assessment)создается группа.

Группы нельзя создавать как отдельные объекты; Группа должна содержать по крайней мере одну оценку. Чтобы создать группу, необходимо сначала создать оценку для помещения в группу.

### <a name="what-to-know-when-working-with-groups"></a>Что нужно знать при работе с группами

- Имена групп должны быть уникальными в пределах организации.
- Группы не имеют свойств безопасности. Все разрешения связаны с оценками.
- После добавления оценки в группу изменение группы невозможно.
- Связанные элементы управления оценки при различных оценках в пределах одной группы автоматически обновляются по завершении.
- При добавлении новой оценки в существующую группу Общие сведения из оценок в этой группе копируются в новую оценку.
- Группы могут содержать оценки для одного и того же сертификата или регламентирования, но каждая группа может содержать только одну оценку для определенной связи продукта с сертификатом. Например, Группа не может содержать две оценки для Office 365 и NIST CSF. Группа может содержать несколько оценок для одного и того же продукта только в том случае, если соответствующая сертификация или регламентирование для каждого из них отличаются.
- При удалении оценки нарушается связь между этой оценкой и группой.
- Удаление групп невозможно.
- При внесении изменений в улучшение, которое отображается в нескольких группах, это изменение отражается во всех экземплярах этого действия по улучшению.

## <a name="create-an-assessment"></a>Создание оценки

Чтобы создать оценку в показателе соответствия требованиям, выполните указанные ниже действия.

1. На странице "оценки" выберите **добавить оценку**. В большой всплывающей области появится мастер оценки.

2. **Выберите шаблон:** Выберите шаблон, который будет использоваться в качестве основы для оценки. Вы можете выбрать шаблон готов к использованию или шаблон, который вы изменили или создали. Выберите переключатель рядом с выбранным шаблоном, а затем нажмите кнопку **Далее**.

> [!NOTE]
> В этой [статье приведены инструкции по созданию и изменению шаблонов](working-with-compliance-manager.md#templates), процесс обработки в диспетчере соответствия требованиям.

3. **Имя и Группа:** Введите имя оценки в поле **имя оценки** . Имена оценки должны быть уникальными в пределах групп. Если имя оценки соответствует имени другой оценки в любой группе, появится сообщение об ошибке с предложением создать другое имя.

4. Назначение оценки группе. Можно выполнить одно из следующих действий:
    - Выберите **использовать существующую группу** , чтобы назначить ее уже созданной вами группе. также
    - Выберите **создать новую группу** , чтобы создать новую группу и назначить ей эту оценку. Определите имя для группы и введите его в поле под переключателем.

5. **Проверка и завершение:** На последнем экране мастера отображается шаблон, имя и группа, выбранные для оценки. Любой из этих параметров можно изменить в ссылках на экране, чтобы вернуться к соответствующим действиям в мастере. Завершив настройку параметров, выберите **создать оценку**.

6. Следующий экран подтверждает, что вы успешно создали новую оценку. Нажмите кнопку **Готово** , чтобы закрыть мастер, и на экране появится страница сведений о новой оценке.

Если после выбора параметра **создать оценку**отображается **неудачный** экран, нажмите кнопку **повторить еще раз** , чтобы повторно создать оценку.

## <a name="delete-an-assessment"></a>Удаление оценки

При удалении оценки она удаляется из списка на странице "оценки". **Удаление оценки необратимо; его нельзя вернуть.** Если вы еще раз хотите повторить оценку, его необходимо создать повторно с нуля. Если действия по улучшению в оценке не отображаются в какой-либо другой оценке, они будут удалены при удалении оценки. Рекомендуется экспортировать отчет об оценке до окончательного удаления.

Чтобы удалить оценку, выполните указанные ниже действия.

1. На странице оценки выберите оценку, которую вы хотите удалить, чтобы открыть страницу сведений оценки.
2. Выберите **удалить оценку** в правом верхнем углу экрана.
3. Откроется окно с запросом подтверждения окончательного удаления оценки. Нажмите кнопку **удалить оценку** , чтобы закрыть окно. Вы получите окно подтверждения, которое было удалено из оценки соответствия требованиям.

Если удалить единственную оценку в группе, эта группа также будет удалена из оценки соответствия требованиям.

## <a name="monitor-assessment-progress-and-controls"></a>Отслеживание хода оценки и элементов управления

Каждая оценка содержит страницу сведений, которая дает представление о ходе выполнения оценки. На странице отображается ход выполнения работ по завершении элементов управления и состояние проверки ключевых действий улучшения в пределах этих элементов управления.

### <a name="overview-tab"></a>Вкладка "Обзор"

Вкладка Обзор содержит график, отражающий процент завершения оценки. Этот график содержит разбивку баллов от действий, которыми вы владеете, и баллов от действий, принадлежащих корпорации Майкрософт, поэтому вы можете узнать, сколько баллов необходимо для выполнения оценки.

Основные действия по улучшению для элементов управления в оценке перечислены в порядке достижения максимальной возможного влияния на заработать очков. В связанном графе представлены сводные сведения о состоянии тестирования действий по улучшению, позволяющие быстро оценить, что было протестировано и что необходимо сделать.

Чтобы получить доступ к отдельным действиям по улучшению, перейдите на вкладку элементы управления.

### <a name="controls-tab"></a>Вкладка с элементами управления

На вкладке элементы управления отображаются подробные сведения для каждого элемента управления, сопоставленного с оценкой. На диаграмме с **разбивкой состояния элемента управления** отображается состояние элементов управления в семействе, поэтому вы можете быстро увидеть, какие группы элементов управления требуют внимания.

Под диаграммой в таблице перечислены подробные сведения о каждом элементе управления в рамках оценки. Элементы управления группируются по семействам элементов управления. Разверните каждое имя семейства, чтобы отобразить отдельные элементы управления, которые он содержит. Сведения, перечисленные для каждого элемента управления, включают:

- **Заголовок элемента управления**
- **Состояние**: отражает состояние тестирования действий улучшения в элементе управления 
    - **Пройдено** — все действия по улучшению имеют состояние проверки "передано" или по крайней мере один из них, а оставшаяся часть "вне области"
    -  **Failed** — по крайней мере одно действие улучшения имеет состояние проверки "Failed"
    - **Нет** — все действия по улучшению не тестировались
    - **За пределами области** — все действия по улучшению выходят за рамки этой оценки
    - **В ходе выполнения действия по** улучшению имеют состояние, отличное от приведенного выше, которое может включать "в данный процесс", "частичный кредит" или "не удается найти"
- **Идентификатор элемента**управления: идентификационный номер элемента управления, назначенный соответствующим нормативным, стандартным или политикой
- Получено **очков**: количество баллов, полученных при выполнении действий, из общего числа достижимых очков 
- **Ваши действия**: количество выполненных действий из общего числа действий, которые необходимо выполнить
- **Действия Майкрософт**: количество действий, выполненных корпорацией Майкрософт 

Чтобы просмотреть сведения об элементе управления, выберите его из строки в таблице. На странице "сведения об элементе управления" показан график, указывающий состояние тестов действий в пределах этого элемента управления. В таблице под диаграммой показаны основные действия по улучшению для этого элемента управления.

Выберите действие улучшения в списке, чтобы перейти на страницу подробных сведений действия по улучшению. На страницах со сведениями отображаются состояние тестирования, заметки о реализации и запуск в рекомендуемом решении.

## <a name="export-an-assessment-report"></a>Экспорт отчета о оценке

Вы можете экспортировать оценку в файл Excel для заинтересованных лиц в Организации, а также для внешних аудиторов и стабилизаторов, [выполнив следующие инструкции](working-with-compliance-manager.md#reports). Для экспорта отчета необходимо посетить диспетчер соответствия требованиям.

Отчет является моментальным снимком оценки на дату и время экспорта. Он содержит сведения об элементах управления, управляемых вами и корпорацией Майкрософт, в том числе о состоянии реализации, тестовой дате, результатах тестов и ссылках на отправленные документы свидетельств.
