# <a name="work-with-improvement-actions-in-compliance-score-preview"></a>Работать с действиями по улучшению в показателе соответствия требованиям (Предварительная версия)

**В этой статье:** В этой статье объясняется, как **управлять рабочим процессом соответствия требованиям** с помощью действий по улучшению. Узнайте, как **назначать действия по улучшению** для реализации и тестирования, назначать их оценкам для выполнения и экспортировать **отчеты**.

## <a name="manage-compliance-workflows-with-improvement-actions"></a>Управление рабочими процессами соответствия с действиями по улучшению

Действия по улучшению централизует действия по обеспечению соответствия требованиям. Каждое действие по улучшению приводит к подробному использованию рекомендаций по обеспечению защиты данных и стандартам. Для выполнения реализации и тестирования можно назначить действия пользователям в вашей организации. Вы также можете хранить документацию, заметки и обновления состояния записи в действии улучшения.

Все улучшения перечислены на странице действия по улучшению. Узнайте больше о [том, как фильтровать представление действий по улучшению и интерпретировать](compliance-score-setup.md#improvement-actions-page)состояния.

## <a name="improvement-actions-details-page"></a>Страница сведений о действиях по улучшению

В каждом действии улучшения отображается страница сведений о текущем состоянии, а также соответствующие стандарты и нормативные требования. Подробное руководство по реализации включает ссылку " **запустить** ", в которой вы можете перейти к соответствующему решению для реализации. Документация по реализации и тестированию можно присоединить непосредственно к странице сведений действия по улучшению.

Чтобы просмотреть страницу сведений действия по улучшению:

1. Перейдите на страницу действий по улучшению.
2. Выберите строку предполагаемого действия по улучшению, чтобы открыть страницу со сведениями.

Вы можете легко просмотреть следующее или предыдущее действие по улучшению в списке, щелкнув стрелку вверх или вниз в правом верхнем углу экрана. Если вы фильтруете список на странице действия по улучшению, перемещение вверх или вниз позволяет перейти к следующему элементу в этом отфильтрованном списке.

## <a name="assign-improvement-actions"></a>Назначение действий по улучшению

Чтобы начать работу над действием по улучшению, вы можете выполнить работу самостоятельно или назначить его другому пользователю. Назначенный пользователь может иметь следующие значения:

- владелец бизнес-политики;
- специалист по внедрению ИТ;
- Другой сотрудник с ответственностью за выполнение задачи

Определив соответствующий уполномоченный объект, убедитесь, что они содержат достаточно [ролей в показателе соответствия требованиям](compliance-score-setup.md#set-user-permissions-and-assign-roles) (администратор соответствия требованиям, администратор данных соответствия требованиям, администратор безопасности или глобальный администратор) для выполнения работы, а затем выполните следующие действия:

1. На странице "сведения о действиях по улучшению" выберите **изменить состояние** в левом верхнем углу экрана.

2. В всплывающей панели изменить состояние выберите поле **назначено** , чтобы показать список **предлагаемых** пользователей. Вы можете выбрать пользователя из списка или ввести адрес электронной почты человека, которому вы хотите назначить его.

3. Нажмите кнопку **сохранить и закрыть**. Назначенному пользователю будет отправлено сообщение электронной почты о том, что ему назначено действие улучшения, и они могут открыть действие улучшения на их панели мониторинга.

Затем назначенный пользователь может выполнить Рекомендуемые действия.

## <a name="perform-work-and-store-documentation"></a>Выполнение работы и сохранение документации

Вы можете отправлять файлы и заметки, связанные с реализацией и тестированием, непосредственно в раздел **Notes and Documentation** . Эта среда является безопасным централизованным репозиторием, который поможет продемонстрировать степень удовлетворенности элементов управления для соответствия стандартам и нормативным требованиям. Любой пользователь с доступом только для чтения может читать содержимое этого раздела. Только пользователи с правами на редактирование могут отправлять и загружать файлы, а также вводить и редактировать заметки.

Поля в разделе **Notes and Documentation** включают:

### <a name="uploaded-documents"></a>Отправленные документы

- Выберите **Управление документами** , чтобы отправить нужные файлы.
- Когда откроется всплывающая область Управление документами, выберите **Добавить документ**, а затем выберите файл из вашей системы. Допустимые типы файлов:
    - Документы (doc, XLS, PPT, txt, PDF)
    - Изображения (JPG, PNG)
    - Видео (. МКВ)
    - Сжатые файлы (ZIP, RAR)
- Когда файл разрешается в области, нажмите кнопку **Закрыть**, который автоматически сохраняет вложенный файл. После этого вы увидите файл, указанный в **передаваемых документах**.
- Чтобы скачать или удалить документ, выберите **Управление документами** в списке документов. В раскрывающейся области щелкните строку документа, чтобы выделить ее, а затем выберите **загрузить** или **Удалить**.

### <a name="implementation-notes-test-notes-and-additional-notes"></a>Примечания по реализации, тестовые заметки и дополнительные примечания

- Чтобы добавить примечания в любое из этих трех полей, выберите **изменить заметки реализации** под любым из этих полей.
- Когда откроется раскрывающаяся панель, введите примечания в текстовое поле, а затем нажмите кнопку **сохранить и закрыть**.
- Чтобы изменить заметки, выберите **изменить заметки реализации**, внесите изменения, а затем нажмите кнопку **сохранить и закрыть**.

В полях Notes нет ограниченных символов. Рекомендуется хранить заметки, чтобы их можно было легко просматривать и редактировать на странице "сведения о действиях по улучшению".

## <a name="change-improvement-action-status"></a>Изменение состояния действия по улучшению

Вы можете записать состояние и дату реализации, а также состояние и дату тестирования для каждого действия по улучшению. Поля " **Реализация** " и " **Состояние тестирования** " могут редактироваться любым пользователем с разрешениями на редактирование, а не только назначенным пользователем.

Чтобы изменить состояние действия улучшения, выберите **изменить состояние** в левом верхнем углу страницы сведений. Ниже перечислены доступные поля и параметры состояния.

- **Состояние реализации**
    - **Не реализовано** — действие еще не реализовано
    - **Реализованное** действие Реализовано
    - **Альтернативная реализация** — выберите этот вариант, если вы использовали другие сторонние средства или выполняли другие действия, не включенные в рекомендации Майкрософт
    - **Запланированное** действие запланировано для реализации
    - **Не в области** действия — действие не относится к вашей организации и не влияет на вашу оценку.
- **Дата реализации**: доступна для выбора, когда состояние реализации — "реализовано" или "альтернативная реализация"
- **Состояние проверки**: доступно для выбора, когда состояние реализации — "реализовано" или "альтернативная реализация":
    - **Не оценено** — действие не тестировалось
    - **Переданная**реализация проверена средством оценки
    - **Сбой при низком риске** — сбой тестирования, недостаток риска
    - **Сбой среднего риска** при неудачном тестировании, умеренный риск
    - **Высокий риск** : сбой тестирования, высокий риск
    - **Не входит в область** — действие не является областью для оценки и не влияет на оценку.
- **Тестовая дата**: переключитесь в меню "Календарь", чтобы выбрать дату.

Общие действия синхронизируются между группами. При двух различных оценках, выполняемых вами в одной группе, любые обновления, вносимые в сведения о реализации действия или состояние, будут автоматически синхронизированы с одним действием в любой другой оценке в группе. Эта синхронизация позволяет реализовать одно действие по улучшению и отвечать нескольким требованиям в отношении нескольких нормативных требований.

## <a name="assign-improvement-action-to-assessor-for-completion"></a>Назначение действия по улучшению для оценки завершения

После выполнения работ, проведения тестирования и отправки свидетельства необходимо назначить действие по улучшению для проверки.

Ниже перечислены типы оценивающих.

- Внутренние средства оценки, выполняющие проверку элементов управления в Организации
- Внешние средства проверки, которые проверяют, проверяют и сертифицировать соответствие требованиям, например сторонние независимые организации, которые поднимаются аудиту облачных служб Майкрософт

Проверяющий проверяет работу и проверяет документацию, а также выбирает соответствующее состояние проверки.

**Если для состояния тестирования задано значение "передано"**: действие завершено, а достигнуто максимальное количество баллов. Затем баллы подсчитываются в общий рейтинг соответствия требованиям.

**Если для состояния тестирования задано значение "Failed"**: действие не соответствует требованиям, и его можно назначить соответствующему пользователю для дополнительной работы.

## <a name="export-a-report"></a>Экспорт отчета

Выберите **Экспорт** в левом верхнем углу экрана, чтобы скачать лист Excel, содержащий все действия по улучшению, а также категории фильтров, показанные на странице действия по улучшению.
