# <a name="data-protection-impact-assessments-guidance-for-data-controllers-using-microsoft-office-365"></a>Оценка влияния защиты данных: Руководство для контроллеров данных с помощью Microsoft Office 365 

В разделе Общие данные защиты нормы (GDPR), контроллеры данных являются обязательными для подготовки оценки влияния данных защиты (DPIA) для обработки операций, которые являются «может привести к высокого риска для прав и freedoms физических лиц.» Нет ничего непосредственно в Microsoft Office 365, обязательно потребует создания DPIA контроллером данных с помощью его.  Вместо этого DPIA необходим будет зависеть сведений и контексте как контроллера данных развертывает, настраивает и использует Office 365.

Этот документ предназначен для предоставления данных контроллеров со сведениями об Office 365, помогающие им для определения необходимости DPIA и если это так, что сведения для включения. Применяется к Office 365 приложений и служб, включая, но не ограничиваясь Exchange Online, SharePoint Online, OneDrive для бизнеса, Yammer Скайп для бизнеса и Power BI.  (См, например, таблицы 1 и 2 данных теме запроса руководство по Office 365).

### <a name="part-2--contents-of-a-dpia"></a>Часть 2. Содержимое DPIA 
Статья 35(7) требует указания, что оценка влияния защиты данных в целях обработки и систематического описание воображаемого обработки.  В корпорации Майкрософт DPIAs таких систематического описание включает в себя факторы такие, как типы данных обработки, как узнать срок хранения данных, где находится и передачу данных, а какие сторонние производители могут иметь доступ к данным.  Кроме того необходимо включить DPIA: 
- оценку необходимости и пропорциональности операций обработки по отношению к целям;  
- оценку рисков для прав и свобод физических лиц; и  
- меры реагирования на риски, включая меры безопасности и механизмы для обеспечения защиты персональных данных и демонстрации соблюдения этого Регламента с учетом прав и законных интересов субъектов данных и других заинтересованных лиц.
