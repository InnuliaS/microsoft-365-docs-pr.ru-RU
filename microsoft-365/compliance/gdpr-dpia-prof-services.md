# <a name="data-protection-impact-assessments-guidance-for-data-controllers-using-microsoft-professional-services"></a>Оценка влияния защиты данных: Руководство для контроллеров данных с помощью служб продуктов корпорации Майкрософт

## <a name="introduction-to-microsoft-professional-services"></a>Общие сведения о профессиональных услугах Майкрософт

Службы Microsoft Professional включает в себя различные группы технического архитекторов, инженеров, консультантов и поддержки Майкрософт предлагает на критически корпорации Майкрософт увеличение эффективности работы клиентов для работы и достижения. Узнайте больше о Microsoft Professional Services здесь (<https://www.microsoft.com/microsoftservices/professional_services.aspx>) и последовательно выбрав пункты в разделе службы Microsoft Professional в центре управления безопасностью Microsoft (<https://www.microsoft.com/trustcenter/cloudservices/commercialsupport>).

Службы Microsoft Professional серьезно его обязательства в разделе Общие данные защиты нормы (GDPR). Предоставляет сведения, приведенные в этом документе приведены сведения о том, как поддержка корпорации Майкрософт и консультационные услуги, которые можно использовать при подготовке оценки влияния защиты данных (DPIAs) в разделе GDPR.

### <a name="introduction-to-dpias"></a>Общие сведения о DPIAs

В разделе Общие данные защиты нормы (GDPR), контроллеры данных являются обязательными для подготовки DPIA для обработки операций, которые являются «может привести к высокого риска для прав и freedoms физических лиц.» Нет ничего присущи в службах Professional, обязательно потребует создания DPIA контроллером данных с помощью его. Вместо этого DPIA необходим будет зависеть сведений и контекст типа служб и как контроллера данных использует профессиональных услуг.

В этом документе предназначен для предоставления данных контроллеров со сведениями о службах Professional, с помощью которых позволяет определить ли необходимые DPIA и, если это так, что приведены сведения о для включения.

## <a name="part-2--contents-of-a-dpia"></a>Часть 2. Содержимое DPIA

Статья 35(7) требует, чтобы оценка влияния на защиту данных определяла цели и системное описание предполагаемой обработки. Системное описание всесторонней оценки DPIA может включать такие факторы, как типы обрабатываемых данных, продолжительность хранения данных, места размещения и передачи данных и третьи стороны, которым может быть предоставлен доступ к этим данным. Кроме того, DPIA должна включать:

-   оценку необходимости и пропорциональности операций обработки по отношению к целям;

-   оценку рисков для прав и свобод физических лиц; и

-   меры реагирования на риски, включая меры безопасности и механизмы для обеспечения защиты персональных данных и демонстрации соблюдения этого Регламента с учетом прав и законных интересов субъектов данных и других заинтересованных лиц.
