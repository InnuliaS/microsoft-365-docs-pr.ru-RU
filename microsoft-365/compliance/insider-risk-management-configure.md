# <a name="get-started-with-insider-risk-management-preview"></a>Начало работы с управлением рисками для оценки безопасности (Предварительная версия)

Используйте политики управления рисками для оценки рискованных действий и средств управления, которые будут выполнять действия с оповещениями о рисках в Организации. Для получения дополнительных сведений о том, как политики риска для оценки могут помочь в управлении рисками в Организации, ознакомьтесь с разкрывающимся [руководством "Управление рисками в Microsoft 365](insider-risk-management.md)".

>[!IMPORTANT]
>Решение по управлению рисками для Microsoft 365 для оценки безопасности предоставляет возможность клиента для упрощения внутреннего управления на уровне пользователя. Администраторы уровня клиента могут настроить разрешения на предоставление доступа к этому решению для участников Организации и Настройка соединителей данных в центре соответствия требованиям Microsoft 365 для поддержки идентификации на уровне пользователя. рискованные действия. Клиенты могут получить подробные сведения о поведении, символе или материале о производительности, связанном с трудовым занятием, могут рассчитываться администратором и предоставляться для других пользователей в Организации.

## <a name="before-you-begin"></a>Перед началом работы

Прежде чем приступить к управлению рисками для оценки, необходимо подтвердить свою подписку на Microsoft 365. Пользователи, включенные в политики управления рисками для участников программы предварительной оценки, должны иметь лицензию на соответствие Microsoft 365/с и быть включены в подписку Microsoft 365 о.

Если у вас нет существующего плана Microsoft 365 корпоративный, и вы хотите испытать ознакомительную версию управления рисками, вы можете [Добавить Microsoft 365](https://docs.microsoft.com/office365/admin/try-or-buy-microsoft-365) в существующую подписку на Office 365 или [зарегистрироваться для получения ознакомительной](https://www.microsoft.com/microsoft-365/enterprise) версии Microsoft 365 Enterprise ~.

Выполните указанные ниже действия, чтобы настроить и использовать управление рисками для участников в организации Microsoft 365:

## <a name="step-1-required-enable-permissions-for-insider-risk-management"></a>Шаг 1 (обязательный): Включение разрешений для управления рисками для оценки безопасности

Существует четыре роли разрешений, которые используются для настройки и управления управлением рисками для оценки. Чтобы продолжить выполнение этих действий, администратору клиента необходимо сначала назначить роль **администратора "Управление рисками"** .

| **Роль** | **Разрешения роли** |
| ---- | ---------------- |
| **Администратор управления рисками для участников** | Создание, чтение, обновление и удаление политик управления рисками для оценки <br> Создание, чтение, обновление и удаление разрешений и ролей управления рисками для участников |
| **Аналитики управления рисками для участников** | Доступ ко всем оповещениям, обращениям и уведомлениям об управлении рисками для оценки |
| **Исследования по управлению рисками для участников** | Доступ ко всем сообщениям управления рисками для оценки, обращениями, уведомлениями и проводником содержимого для всех случаев. |
| **Средство просмотра управления рисками для оценки** | Доступ только для просмотра ко всем оповещениям управления рисками для оценки, обращениям, уведомлениям и проводнику содержимого для всех случаев. |

В зависимости от структуры группы управления соответствием, у вас есть два варианта разрешений для настройки ролей для управления функциями управления рисками для предварительной оценки. Выберите один из следующих параметров управления ролями при настройке управления рисками для участников:

1. **Использование группы ролей "Управление рисками" по умолчанию**: Используйте эту группу ролей, чтобы управлять управлением рисками для вашей организации, добавляя все учетные записи пользователей для назначенных администраторов, аналитик и изучений в одну группу. Эта группа ролей содержит все роли разрешений по управлению рисками для оценки. Это самый простой способ быстро начать работу с управлением рисками для оценки и подходит для организаций, которым не требуются отдельные разрешения, определенные для отдельных групп пользователей.
2. **Создайте различные группы для различных ролей управления**: для организаций, которым требуется разделять разрешения для настройки и управления управлением рисками для оценки безопасности, необходимо создать новые группы ролей и назначить соответствующие роли и пользователей новым группам. Например, если вы хотите использовать различные разрешения для анализа и исследования управления рисками, вам потребуется создать группу для аналитик и отдельную группу для исследований. Для каждой группы вы назначаете необходимые роли для этих обязанностей, а затем добавляете пользователей, которые должны быть назначены группе.

Если вы решили настроить различные группы для разных ролей, используйте приведенную ниже таблицу, чтобы назначить необходимые роли каждой группе ролей:

| **Пример группы** | **Обязательные роли** |
| ---- | ---------------- |
| **Администраторы** | Роль *администратора по управлению рисками для участников* |
| **Эксперт** | Роль *консультантов по управлению рисками* <br> Роль *управления делами* |
| **Расисследования** | Роль *расследований по управлению рисками для участников* <br> Роль *управления делами* |

### <a name="option-1-add-users-to-the-insider-risk-management-role-group"></a>Вариант 1: Добавление пользователей в группу ролей "Управление рисками"

Если вы хотите использовать одну группу ролей для всех пользователей, настроив и управляя управление рисками для предварительной оценки, выполните следующие действия, чтобы добавить пользователей в группу ролей " **Управление рисками** " по умолчанию:

1. Войдите в [https://protection.office.com/permissions](https://protection.office.com/permissions) систему, используя учетные данные для учетной записи администратора в организации Microsoft 365.

2. В центре безопасности и соответствия требованиям Microsoft Office 365 перейдите к разделу **разрешения**. Выберите ссылку для просмотра ролей в Office 365 и управления ими.

3. Выберите группу ролей *Управление рисками для участников* , а затем выберите пункт **изменить группу ролей**.

4. Выберите **выбрать элементы** в левой области навигации, а затем нажмите кнопку **изменить**.

5. Нажмите кнопку **Добавить** , а затем установите флажок для всех пользователей, которых вы хотите добавить в группу ролей.

6. Нажмите кнопку **Добавить**, а затем нажмите кнопку **Готово**.

7. Нажмите кнопку **сохранить** , чтобы добавить пользователей в группу ролей. Нажмите кнопку **Закрыть** , чтобы завершить процедуру.

### <a name="option-2-create-a-new-role-group"></a>Вариант 2: создание новой группы ролей

Если вам нужно создать отдельные группы ролей для разных ролей управления, выполните следующие действия для создания каждой новой группы:

1. Войдите в [https://protection.office.com/permissions](https://protection.office.com/permissions) систему, используя учетные данные для учетной записи администратора в организации Microsoft 365.

2. В центре безопасности и соответствия требованиям Microsoft Office 365 перейдите к разделу **разрешения**. Выберите ссылку для просмотра ролей в Office 365 и управления ими.

3. Нажмите кнопку **Создать**.

4. В поле **имя** присвойте новой группе ролей понятное имя. Нажмите кнопку **Далее**.

5. Нажмите **кнопку Выбрать роли** , а затем нажмите кнопку **Добавить**. Установите флажки для ролей, которые необходимо назначить. Например, если эта группа предназначена для аналитик риска, выберите роль " *аналитики управления рисками* " и "роль *управления делами* ", а затем нажмите кнопку **добавить** и **выполнить**. Нажмите кнопку **Далее**.

6. Выберите **пункт Выбрать элементы** и нажмите кнопку **Добавить**. Установите флажок для всех пользователей и групп, для которых нужно создать политики, и управления сообщениями с соответствием политик, а затем нажмите кнопку **Добавить** и **выполнить**. Нажмите кнопку **Далее**.

7. Выберите **создать группу ролей** для завершения.

Дополнительные сведения о группах ролей и разрешениях см [в разделе разрешения в центре соответствия требованиям](../security/office-365-security/protect-against-threats.md).

## <a name="step-2-optional-configure-the-microsoft-365-human-resources-data-connector"></a>Шаг 2 (необязательно): Настройка соединителя данных отдела кадров Microsoft 365

Центр управления рисками для предварительной оценки поддерживает импорт данных пользователей и журналов, импортированных с платформ управления рисками сторонних производителей и платформ управления персоналом. Соединитель данных отдела кадров Microsoft 365 (HR) позволяет получать данные о персонале из CSV-файлов, в том числе даты увольнения пользователей и даты последнего занятости. Эти данные помогают отменять индикаторы оповещений в политиках по управлению рисками и являются важной частью настройки полной ликвидности управления рисками в Организации.

Пошаговое руководство по настройке соединителя Microsoft 365 HR для организации можно посмотреть в статье [Настройка соединителя для импорта данных о HR](import-hr-data.md) . После настройки соединителя кадров вернитесь к этим этапам настройки.

>[!IMPORTANT]
>Если вы планируете настроить политику с помощью функции "Включение и выключение *кражи данных сотрудников* ", вам потребуется настроить соединитель кадров, чтобы использовать функции полного обнаружения сигналов шаблона политики. Если для вашей организации настроено несколько соединителей для отдела кадров, управление рисками для предварительной оценки будет автоматически получать индикаторы со всех соединителей отдела кадров.

## <a name="step-3-optional-configure-data-loss-prevention-dlp-policies"></a>Шаг 3 (необязательно): Настройка политик защиты от потери данных (DLP)

Для определения преднамеренных или случайных раскрытия конфиденциальной информации нежелательным лицам можно использовать политики защиты от несанкционированного доступа. При настройке политики управления рисками для предварительной оценки с помощью шаблона " *утечки данных* " необходимо назначить политике определенную политику защиты от потери данных. Эта политика помогает управлять показателями оповещения для конфиденциальной информации является важной частью настройки всего охвата управления рисками в Организации.

В разделе [Создание, тестирование и Настройка политики DLP](create-test-tune-dlp-policy.md) пошаговое руководство по настройке политик защиты от потери данных в Организации. После настройки политики защиты от потери данных вернитесь к этим этапам настройки.

>[!IMPORTANT]
>Если вы планируете настроить политику с помощью шаблона " *утечки данных* ", необходимо настроить по крайней мере одну политику защиты от потери данных, чтобы использовать функции полного обнаружения сигналов шаблона политики. Если для организации настроено несколько политик защиты от потери данных, необходимо назначить политику управления рисками для предварительной оценки на политику DLP.

## <a name="step-4-required-create-an-insider-risk-management-policy"></a>Шаг 4 (обязательный): Создание политики управления рисками для оценки

К политикам управления рисками для предварительной оценки относятся назначенные пользователи и определены типы индикаторов риска, настроенных для оповещений. Прежде чем действия смогут инициировать оповещения, необходимо настроить политику.

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **политики** .
2. Выберите **создать политику** , чтобы открыть мастер политик
3. На странице **Создание политики для оценки риска** заполните следующие поля:
    - **Name (обязательно)**: введите понятное имя для политики.
    - **Description (необязательно)**: введите описание политики.
    - **Выбор шаблона политики (обязательно)**: выберите один из [шаблонов политики](insider-risk-management-policies.md#policy-templates) , чтобы определить типы индикаторов риска, контролируемые политикой.

    >[!IMPORTANT]
    >Если выбран шаблон " *утечки данных* ", необходимо настроить по крайней мере одну политику защиты от потери данных, которая будет назначена позже в мастере. Если выбран шаблон " *хищение данных о сотрудниках* ", необходимо настроить соединитель кадров, чтобы использовать функции полного обнаружения сигналов шаблона политики.

4. Нажмите кнопку **Далее** , чтобы продолжить.
5. На странице **Пользователи** выберите **Добавить пользователя или группу** , чтобы определить, какие пользователи включены в политику или флажок выбрать **все пользователи и группы с включенной поддержкой почты** . Нажмите кнопку **Далее** , чтобы продолжить.
6. На странице **Указание содержимого для определения приоритета (необязательно)** можно назначить источникам приоритеты для рискованных действий пользователей:
    - **Сайты SharePoint**: выберите пункт **Добавить сайт SharePoint** и выберите Организации SharePoint, для которых необходимо задать приоритет. Например, *"Group1@contoso.sharepoint.com/sites/group1"*.
    - **Тип конфиденциальной информации**: выберите **Добавить тип конфиденциальной информации** и выберите типы чувствительности, которые нужно назначить приоритетам. Например, *"номер банковского счета США"* и *"номер кредитной карты"*.
    - **Метки конфиденциальности**: выберите **Добавить метку конфиденциальности** и выберите метки, для которых требуется задать приоритет. Например, *"конфиденциальный"* и *"секрет"*.
7. Нажмите кнопку **Далее** , чтобы продолжить.
8. На странице **Выбор индикаторов оповещений** вы увидите индикаторы, включенные в шаблон, выбранный для этой политики. Если в начале работы мастера был выбран шаблон " *утечки данных* ", необходимо выбрать политику защиты от потери данных в раскрывающемся списке **Политика защиты от потери** данных.
9. На странице **Выбор окна мониторинга** вы можете определить [условия окна мониторинга](insider-risk-management-policies.md#monitoring-windows) для политики. Настройте окна мониторинга соответствующим образом.
10. Нажмите кнопку **Далее** , чтобы продолжить.
11. На странице " **Проверка** " Проверьте параметры, выбранные для политики. Нажмите кнопку **изменить** , чтобы изменить любое значение политики, или кнопку **послать** , чтобы создать и активировать политику.

После выполнения этих действий для создания первой политики управления рисками для предварительной оценки вы начнете получать оповещения от индикаторов активности через 24 часа. Настройте дополнительные политики по мере необходимости, используя инструкции, приведенные в шаге 4 этой статьи, или действия, описанные в статье [Создание новой политики риска для оценки](insider-risk-management-policies.md#create-a-new-policy).
