# <a name="insider-risk-management-policies-preview"></a>Политики управления рисками для предварительной оценки (ознакомительная версия)

Политики управления рисками для предварительной оценки определяют, какие сотрудники находятся в области действия и какие типы индикаторов риска настроены для оповещений. Вы можете быстро создать политику, которая применяется ко всем пользователям в Организации, или определять отдельных пользователей или группы для управления в политике. Политики поддерживают приоритеты содержимого для фокусировки условий политики на нескольких или определенных Microsoft Teams, на сайтах SharePoint, типах чувствительности данных и метках данных. С помощью шаблонов они включают определенные индикаторы риска и степень их назначения в политике, эффективно определяя вес каждого триггера оповещений в политике. Окна политик позволяют определить период времени, в течение которого политика будет применяться к действиям оповещения, и используется для определения продолжительности политики после активации. Максимальное ограничение политики одновременно относится к пяти активным политикам. Тем не менее, при необходимости можно настроить дополнительные политики, а также активировать и деактивировать политики.

## <a name="policy-dashboard"></a>Панель мониторинга политик

**Панель мониторинга политики** позволяет быстро просматривать политики в Организации и текущие состояния оповещений, связанных с каждой политикой.

- **Имя политики**: имя, назначенное политике в мастере политик.
- **Активные оповещения**: количество активных оповещений для каждой политики.
- **Подтвержденные оповещения**: общее число оповещений, которые привели к случаям из политики за последние 365 дней.
- **Действия, выполняемые с оповещениями**: общее количество оповещений, которые были подтверждены или отклонения за последние 365 дней.
- **Эффективность политики**: процентное отношение, определяемое общим количеством подтвержденных оповещений, разделенных на общее количество действий, выполняемых над оповещениями (это сумма подтвержденных или отклоненных оповещений за последний год).
- **Active**: состояние обращения ( *Да* или *нет*).

## <a name="policy-templates"></a>Шаблоны политики

Шаблоны управления рисками для предварительной оценки — это предопределенные условия политики, определяющие типы показателей риска, отслеживаемых политикой. У каждой политики должен быть шаблон, назначенный в мастере создания политики до создания политики. При создании новой политики риска для оценки можно выбрать один из следующих шаблонов.

### <a name="departing-employee-data-theft"></a>Частичная кража данных сотрудников

Когда сотрудники покидают вашу организацию, существуют определенные индикаторы риска, которые обычно связаны с хищением данных путем их посторонних сотрудников. Этот шаблон политики определяет приоритеты этих индикаторов и обнаружение фокусирований и оповещений в этой области риска. Кража данных для посторонних сотрудников может включать скачивание файлов из SharePoint Online, копирование файлов на переносные устройства, такие как USB-накопители, печать файлов, а также копирование данных в личные облачные службы обмена сообщениями и хранения данных в ближайшем ресигнатион и даты окончания. Этот шаблон определяет приоритеты рисков, связанных с этими действиями, и их взаимоотношения со статусом занятости сотрудника.

>[!IMPORTANT]
>При использовании этого шаблона необходимо настроить соединитель Microsoft 365 HR для периодического импорта сведений о ресигнатион и даты увольнения сотрудников в Организации. Пошаговое руководство по настройке соединителя Microsoft 365 HR для вашей организации представлено в статье [Импорт данных с помощью СОЕДИНИТЕЛЯ HR](import-hr-data.md) .

### <a name="data-leaks"></a>Утечки данных

Защита данных и предотвращение потерь данных является постоянной задачей для большинства организаций, особенно с быстрым увеличением новых данных, созданных сотрудниками, устройствами и службами. Сотрудники могут создавать, хранить и совместно использовать информацию на разных службах и устройствах, которые делают управление утечками данных более сложными и сложными. Утечки данных могут включать случайный общий доступ к информации вне организации или кражи данных с помощью злонамеренного способа. Этот шаблон приоритетно определяет обнаружение подозрительных данных в SharePoint Online, общий доступ к файлам и папкам, копирование файлов на переносные устройства, такие как USB-накопители, печать файлов и копирование данных в личные облачные службы обмена сообщениями и хранилищами.

>[!IMPORTANT]
>При использовании этого шаблона необходимо настроить по крайней мере одну политику защиты от потери данных (DLP) для определения конфиденциальной информации в Организации. В разделе [Создание, тестирование и Настройка политики DLP](create-test-tune-dlp-policy.md) пошаговое руководство по настройке политик защиты от потери данных в Организации.

### <a name="offensive-language-in-email"></a>Оскорбительный язык в сообщении электронной почты

Обнаружение и принятие меры по предотвращению оскорбительных и нежелательных функций является важнейшим компонентом предотвращения риска. Встроенные классификаторы нежелательных языков в Microsoft 365 могут сканировать отправленные сообщения электронной почты из почтовых ящиков Exchange Online в Организации для различных типов проблем с соответствием. Эти классификаторы используют сочетание искусственного интеллекта и ключевых слов для определения языка в сообщениях электронной почты, которые могут нарушить политику защиты от преследований. Используйте этот шаблон, чтобы быстро создать политику, которая использует эти классификаторы для автоматического обнаружения сообщений электронной почты, которые могут считаться оскорбительными или оскорбительными. Центр управления рисками для предварительной оценки использует классификаторы, которые проверяют посланные сообщения электронной почты на англоязычные языковые термины и мнений для оскорбительного языка.

## <a name="monitoring-windows"></a>Окна мониторинга

Окна мониторинга политики позволяют определить периоды времени, которые активируют активацию политики на основе событий и действий для шаблонов политики управления рисками. В зависимости от выбранного шаблона политики доступны следующие окна мониторинга:

- **В области TimeSpan**: доступно для всех шаблонов политик, значение *TimeSpan в области* — это определенное количество дней, в течение которых окно активируется **после** запуска события. Окно активируется в течение 1 – 30 дней после возникновения события запуска для любого пользователя, назначенного политике. Например, вы настроили политику управления рисками для предварительной оценки и установите значение *в диапазоне от TimeSpan* до 30 дней. После настройки политики прошло несколько месяцев, а для одного из пользователей, включенных в политику, возникает событие триггера. Событие триггера активирует *область временного интервала* , а политика активна для этого пользователя в течение 30 дней после возникновения события триггера.
- **Исторические данные TimeSpan**: доступны для всех шаблонов политик, *историческая TimeSpan* — это определенное количество дней, в течение которых окно активируется **до** запуска события. Окно активируется в течение от 0 до 180 дней, пока не будет вызвано событие запуска для любого пользователя, назначенного политике. Например, вы настроили политику управления рисками для оценки и присвоить *историческую TimeSpan* значение 90 дней. После настройки политики прошло несколько месяцев, а для одного из пользователей, включенных в политику, возникает событие триггера. Событие запуска активируется с *предысторией TimeSpan* , а политика собирает исторические действия для этого пользователя в течение 90 дней до события триггера.
- **Окно будущего завершения**: вы увидите этот параметр в общие Preview, но он не применяется ни к каким политикам и будет удален для выпуска этого решения.
- **Окно завершения**работы: вы увидите этот параметр в общие Preview, но он не применяется ни к каким политикам и будет удален для рабочего выпуска этого решения.

## <a name="create-a-new-policy"></a>Создание новой политики

Для создания новой политики управления рисками для предварительной оценки вы будете использовать мастер политик в решении " **Управление рисками** " в центре соответствия требованиям Microsoft 365.

Выполните указанные ниже действия, чтобы создать новую политику.

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **политики** .
2. Выберите **создать политику** , чтобы открыть мастер политик
3. На странице **Создание политики для оценки риска** заполните следующие поля:
    - **Name (обязательно)**: введите понятное имя для политики.
    - **Description (необязательно)**: введите описание политики.
    - **Выбор шаблона политики (обязательно)**: выберите один из [шаблонов политики](insider-risk-management-policies.md#policy-templates) , чтобы определить типы индикаторов риска, контролируемые политикой.

    >[!IMPORTANT]
    >Если выбран шаблон " *утечки данных* ", необходимо настроить по крайней мере одну политику защиты от потери данных, которая будет назначена позже в мастере. Если выбран шаблон " *хищение данных о сотрудниках* ", необходимо настроить соединитель кадров, чтобы использовать функции полного обнаружения сигналов шаблона политики.

4. Нажмите кнопку **Далее** , чтобы продолжить.
5. На странице **Пользователи** выберите **Добавить пользователя или группу** , чтобы определить, какие пользователи включены в политику или флажок выбрать **все пользователи и группы с включенной поддержкой почты** . Нажмите кнопку **Далее** , чтобы продолжить.
6. На странице **Указание содержимого для определения приоритета (необязательно)** можно назначить источникам приоритеты для рискованных действий пользователей:
    - Сайты SharePoint: выберите пункт **Добавить сайт SharePoint** и выберите Организации SharePoint, для которых необходимо задать приоритет. Например, *"Group1@contoso.sharepoint.com/sites/group1"*.
    - Тип конфиденциальной информации: выберите **Добавить тип конфиденциальной информации** и выберите типы чувствительности, которые нужно назначить приоритетам. Например, *"номер банковского счета США"* и *"номер кредитной карты"*.
    - Метки конфиденциальности: выберите **Добавить метку конфиденциальности** и выберите метки, для которых требуется задать приоритет. Например, *"конфиденциальный"* и *"секрет"*.
7. Нажмите кнопку **Далее** , чтобы продолжить.
8. На странице **Выбор индикаторов оповещений** вы увидите индикаторы, включенные в шаблон, выбранный для этой политики. Если в начале работы мастера был выбран шаблон " *утечки данных* ", необходимо выбрать политику защиты от потери данных в раскрывающемся списке **Политика защиты от потери** данных.
9. На странице **Выбор окна мониторинга** вы можете определить [условия окна мониторинга](insider-risk-management-policies.md#monitoring-windows) для политики. Настройте окна мониторинга соответствующим образом.
10. Нажмите кнопку **Далее** , чтобы продолжить.
11. На странице " **Проверка** " Проверьте параметры, выбранные для политики. Нажмите кнопку **изменить** , чтобы изменить любое значение политики, или кнопку **послать** , чтобы создать и активировать политику.

## <a name="update-a-policy"></a>Обновление политики

Чтобы обновить существующую политику управления рисками для предварительной оценки, воспользуйтесь мастером политики в решении по **управлению рисками** в центре соответствия требованиям Microsoft 365.

Выполните следующие действия, чтобы управлять существующей политикой:

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **политики** .
2. В панели мониторинга политика выберите политику, которую вы хотите изменить.
3. На странице "сведения о политике" выберите **изменить политику** .
4. В мастере политики невозможно изменить следующие поля:
    - **Name**: понятное имя политики.
    - **Выберите стратегия**: шаблон, используемый для определения типов индикаторов риска, отслеживаемых политикой.
5. Введите новое описание политики в поле **Описание** . Нажмите кнопку **Далее** , чтобы продолжить.
6. На странице **Пользователи** выберите **Добавить пользователя или группу** , чтобы определить, какие пользователи включены в политику или флажок выбрать **все пользователи и группы с включенной поддержкой почты** . Нажмите кнопку **Далее** , чтобы продолжить
7. На странице **Указание содержимого для определения приоритета (необязательно)** обновите источники, чтобы определить приоритет для рискованных действий пользователей:

    - Метки конфиденциальности: выберите **Добавить метку конфиденциальности** и выберите метки, для которых требуется задать приоритет. Например, *"конфиденциальный"* и *"секрет"*.
8. Нажмите кнопку **Далее** , чтобы продолжить.
9. На странице **Выбор индикаторов оповещений** вы увидите индикаторы, включенные в шаблон, выбранный для этой политики. Если в начале работы мастера был выбран шаблон " *утечки данных* ", необходимо выбрать политику защиты от потери данных в раскрывающемся списке **Политика защиты от потери** данных.
10. На странице **Выбор окна мониторинга** вы можете определить [условия окна мониторинга](insider-risk-management-policies.md#monitoring-windows) для политики. Настройте окна мониторинга соответствующим образом.
11. На странице " **Проверка** " Проверьте параметры, выбранные для политики. Нажмите кнопку **изменить** , чтобы изменить любое значение политики, или кнопку **послать** , чтобы обновить и активировать изменения в политике.

## <a name="delete-a-policy"></a>Удаление политики

>[!NOTE]
>Удаление политики не приводит к удалению активных или архивированных оповещений, созданных из политики.

Чтобы удалить существующую политику безопасности для предварительной оценки, выполните указанные ниже действия.

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **политики** .
2. В панели мониторинга политика выберите политику, которую вы хотите изменить.
3. Нажмите кнопку **Удалить** на панели инструментов панели мониторинга.
4. В диалоговом окне **Удалить** нажмите кнопку **Да** , чтобы удалить политику, или кнопку **Отмена** , чтобы закрыть диалоговое окно.
