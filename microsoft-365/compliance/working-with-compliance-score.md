# <a name="working-with-microsoft-compliance-score-preview"></a>Работа со оценки соответствия требованиям Майкрософт (Предварительная версия)

В этой статье объясняется, как работать с ключевыми элементами оценки соответствия требованиям. Вы узнаете, как использовать **действия по улучшению** для управления рабочими процессами соответствия требованиям. Вы также узнаете, как использовать сведения на страницах **решений** и **оценки** , а также как создавать отчеты.

## <a name="manage-your-workflow-with-improvement-actions"></a>Управление рабочим процессом с помощью действий по улучшению

**Действия по улучшению** централизует действия по обеспечению соответствия требованиям. Каждое действие по улучшению приводит к подробному использованию рекомендаций по обеспечению защиты данных и стандартам. Для выполнения реализации и тестирования можно назначить действия пользователям в вашей организации. Вы также можете хранить документацию, заметки и обновления состояния записи в действии улучшения.

## <a name="view-your-improvement-actions"></a>Просмотр действий по улучшению

На панели мониторинга "Оценка соответствия требованиям" отображаются **основные действия по улучшению**— с самыми доступными точками, которые устраняют наиболее важные проблемы.

Чтобы просмотреть все действия по улучшению на странице **действия по улучшению** , перейдите на вкладку **действия по улучшению** на панели мониторинга. Или выберите **Просмотреть все действия по улучшению** под списком ключевых действий по улучшению на панели мониторинга.

Если у вас есть длинный список действий, может быть полезно отфильтровать представление. Выберите пункт **Фильтр** в правом верхнем углу списка действия. Когда откроется раскрывающаяся панель **фильтры** , выберите критерии на основе нормативных требований, стандартов, решений и групп. Кроме того, вы можете настроить представление, выбрав **группу** в правом верхнем углу. В раскрывающемся меню выберите для просмотра по группе, решению, категории, типу действия или состоянию.

Представление по умолчанию для этой страницы не показывает действия по улучшению с **пройденным**состоянием тестирования. Чтобы просмотреть прошедшее тестирование действия, установите флажок **переданное** поле в раскрывающейся панели **фильтры** . Только действия со статусом проверки **переданных** на ваш счет.

На странице действия по улучшению отображаются следующие точки данных для каждого действия по улучшению:

- **Влияние на рейтинг**: точки, на которые увеличивается общий рейтинг при выполнении действия
- **Нормативы**: нормативный или стандартный, относящийся к действию
- **Группа**: группа, которой назначено действие
- **Решения**: решение, с помощью которого можно выполнить действие
- **Оценки**: Оценка (при которой элементы управления упорядочиваются в соответствии с определенным показателем соответствия требованиям), в котором находится действие
- **Категории**: связанная Категория защиты данных (например, защита информации, Управление устройствами и т. д.)
- **Состояние тестирования**:
    - **Нет** — обновление состояния не записано
    - **Не оценено** , что тестирование не начато
    - **Не в области** — исключен из расчета оценки соответствия требованиям и не приводит к увеличению оценки.
    - **Частично протестированное** тестирование еще не завершено
    - **Сбой при высоком риске** — не удалось выполнить тестирование реализации, а также риск несоответствия требованиям в соответствии со стандартом High
    - **Успешное** тестирование реализации
- На **достижение**: показаны баллы, которые можно получить из максимального объема, который можно получить

### <a name="improvement-actions-details"></a>Сведения о действиях по улучшению

Каждое действие по улучшению содержит страницу сведений. На этой странице представлены подробные инструкции по реализации для выполнения рекомендуемых действий, которые относятся к связанным стандартам и нормативным требованиям, перечисленным под заголовком " **краткий обзор** ".

На странице сведения вы можете запустить рекомендуемое действие. Вы также можете назначить работу другому пользователю, статусу обновления, а также присоединить заметки и документацию.

Чтобы просмотреть страницу сведений действия по улучшению:

1. Перейдите на страницу действий по улучшению.
2. Щелкните или коснитесь в любом месте строки предполагаемого действия улучшения, чтобы открыть страницу со сведениями.

Вы можете легко просмотреть следующее или предыдущее действие по улучшению в списке, щелкнув стрелку вверх или вниз в правом верхнем углу экрана. Если вы отфильтрованы в списке на странице "действия по улучшению", перемещение вверх или вниз приведет к переходу к следующему элементу в этом отфильтрованном списке.

## <a name="assign-improvement-actions"></a>Назначение действий по улучшению

Чтобы начать работу над действием по улучшению, вы можете выполнить работу самостоятельно или назначить его другому пользователю. Назначенный пользователь может иметь следующие значения:

- владелец бизнес-политики;
- специалист по внедрению ИТ;
- Другой сотрудник с ответственностью за выполнение задачи

Когда соответствующий пользователь будет идентифицирован, убедитесь, что в списке соответствия требованиям (администратору соответствия требованиям, администратору данных для обеспечения соответствия требованиям, администратору безопасности или глобальному администратору [) выполняется работа](compliance-score-setup.md#set-user-permissions-and-assign-roles) , а затем выполните указанные ниже действия. 

1. На странице "сведения о действиях по улучшению" выберите **изменить состояние** в левом верхнем углу экрана. 

2. В всплывающей панели изменить состояние щелкните или коснитесь поля **Кому назначено** , которое заполняет список **предлагаемых** пользователей. Вы можете выбрать пользователя из списка или ввести адрес электронной почты человека, которому нужно назначить действие.

3. Нажмите кнопку **сохранить и закрыть** , чтобы завершить назначение. Назначенному пользователю будет отправлено сообщение электронной почты о том, что ему назначено действие улучшения, и они могут открыть действие улучшения на их панели мониторинга.

После этого назначенный пользователь может выполнить Рекомендуемые действия, описанные в разделе инструкции по реализации.

## <a name="perform-work-and-store-documentation"></a>Выполнение работы и сохранение документации

Работая над реализацией, вы можете отправлять файлы и заметки непосредственно в действие улучшения в разделе **Notes and Documentation** .  Эта среда является безопасным централизованным репозиторием, который поможет продемонстрировать степень удовлетворенности элементов управления для соответствия стандартам и нормативным требованиям. Любой пользователь с доступом только для чтения может читать содержимое этого раздела. Только пользователи с правами на редактирование могут отправлять и загружать файлы, а также вводить и редактировать заметки.

Поля в разделе **Notes and Documentation** включают:

**Отправленные документы**

- Выберите **Управление документами** , чтобы отправить нужные файлы.
- Когда откроется всплывающая область Управление документами, выберите **Добавить документ**, а затем выберите файл из вашей системы. Допустимые типы файлов: 
  - Документы (doc, XLS, PPT, txt, PDF)
  - Изображения (JPG, PNG)
  - Видео (. МКВ)
  - Сжатые файлы (ZIP, RAR)
- Когда файл будет разрешаться в области, нажмите кнопку **Закрыть,** чтобы автоматически сохранить вложенный файл. Вы увидите файл, указанный в **передаваемых документах.** 
- Чтобы скачать или удалить документ, выберите **Управление документами** в списке документов. В раскрывающейся области щелкните строку документа, чтобы выделить ее, а затем выберите **загрузить** или **Удалить.**

**Реализация, тестирование и дополнительные заметки**

- Чтобы добавить примечания в любое из этих трех полей, выберите **изменить заметки реализации** под любым из этих полей.
- Когда откроется раскрывающаяся панель, введите примечания в текстовое поле, а затем нажмите кнопку **сохранить и закрыть**.
- Чтобы изменить заметки, выберите **изменить заметки реализации**, внесите изменения, а затем нажмите кнопку **сохранить и закрыть**.

В полях Notes нет ограниченных символов. Рекомендуется хранить заметки, чтобы их можно было легко просматривать и редактировать на странице "сведения о действиях по улучшению".

## <a name="change-improvement-action-status"></a>Изменение состояния действия по улучшению

Вы можете записать состояние и дату реализации, а также состояние и дату тестирования для каждого действия по улучшению. Ниже перечислены доступные поля и параметры состояния.

- **Состояние реализации**: выберите следующие параметры состояния:
    - **Не реализовано** — действие еще не реализовано
    - **Реализованное** действие Реализовано
    - **Альтернативная реализация** — выберите этот вариант, если вы использовали другие сторонние средства или выполняли другие действия, не включенные в рекомендации Майкрософт
    - **Запланированное** действие запланировано для реализации
    - **Не в области** — параметр для действий, не относящихся к вашей организации; Выбор этого состояния исключает действие из оценки; Если снять этот флажок, это действие будет включено в значение "Оценка".
- **Дата реализации**: доступное поле, когда для реализации задано состояние **реализовано** или **альтернативная реализация**; Выбор даты в раскрывающемся списке "Календарь"
- **Состояние тестирования**: выберите следующие параметры:
    - **Не оценено** , что тестирование не начато
    - **Успешное**тестирование реализации
    - **Сбой при низком риске** — сбой тестирования, недостаток риска
    - **Сбой среднего риска** при неудачном тестировании, умеренный риск
    - **Сбой при высоком риске** — сбой тестирования, высокий риск
- **Тестовая дата**: переключитесь в меню "Календарь", чтобы выбрать дату.

> [!NOTE]
> Поля "реализация" и "Состояние тестирования" могут редактироваться любым пользователем с разрешениями на редактирование, а не только пользователем, **назначенным** пользователю.

## <a name="assign-improvement-action-to-assessor-for-completion"></a>Назначение действия по улучшению для оценки завершения

После завершения работы и загрузки свидетельства, следующим шагом является установка состояния и даты реализации, а также назначение действия по улучшению для проверки.

Ниже перечислены типы оценивающих.

- Внутренние средства оценки, выполняющие проверку элементов управления в Организации
- Внешние средства оценки, которые проверяют, проверяют и сертифицировать соответствие требованиям, например сторонние независимые организации, которые поднимаются аудиту облачных служб Майкрософт

Проверяющий проверяет работу и проверяет документацию, а также выбирает соответствующее состояние проверки.

**Если состояние проверки — "передано"**: действие завершено, а полученное количество **баллов** показывает полное количество возможных баллов и подсчитывается к общему показателю соответствия требованиям.

**Если состояние проверки имеет значение любая степень "Failed"**: действие не соответствует требованиям, и может быть назначено подходящему пользователю для дополнительной работы.

## <a name="solutions-page"></a>Страница "решения"

**Страница решений** является еще одной отправной точкой для работы с действиями по улучшению, чтобы увеличить рейтинг. 

Чтобы перейти к странице ваших решений, перейдите на вкладку **решения** на панели мониторинга или выберите **Просмотреть все решения** под **решениями, которые влияют на оценку** в правом верхнем углу панели мониторинга.

На странице Solutions показан общий доступ к общим и потенциальным точкам, упорядоченный по решениям. Просмотр оставшихся баллов и действий по улучшению в этом представлении помогает определить, какие решения требуют более быстрого внимания.

### <a name="filtering-your-solutions-view"></a>Фильтрация представления решений

Чтобы отфильтровать решения, выполните следующие действия:

1. Выберите пункт **Фильтр** в левом верхнем углу списка оценки.
2. В области всплывающие **фильтры** установите флажок рядом с нужным критерием (стандарты и нормативы, решение, тип действия, Группа диспетчера соответствия требованиям, категория).
3. Нажмите кнопку **Apply (применить** ). Область фильтра будет закрыта, и вы увидите отфильтрованное представление.

Вы также можете изменить представление, чтобы просмотреть оценки по группам, продуктам или нормативным данным, выбрав тип группировки в раскрывающемся меню **Группа** над списком оценки.

### <a name="taking-actions-from-the-solutions-page"></a>Выполнение действий со страницы "решения"

На странице "решения" отображаются решения вашей организации, которые подключены к действиям по улучшению. В таблице перечислено влияние каждого решения на общий рейтинг, точки повышения качества, которые допускают и могут быть в этом решении, а также оставшееся количество действий по улучшению, сгруппированных в этом решении, которые могут увеличить рейтинг.

На этом экране можно выполнить два способа действий:

1. В столбце " **оставшиеся** решения" строки предполагаемого решения выберите номер гиперссылки. На экране действия по улучшению отображается отфильтрованное представление, в котором показаны непроверенные действия по улучшению для этого решения.

2. В строке предполагаемого решения в столбце **открыть решение** нажмите кнопку **Открыть**. Вы увидите решение или расположение в центрах безопасности и соответствиях Microsoft 365 и Office 365, где можно выполнить Рекомендуемые действия.

## <a name="assessments-page"></a>Страница "оценки"

На странице "оценки" перечислены оценки, выбранные для отслеживания в Организации. Знаменатель показателя соответствия определяется всеми отслеживаемыми оценками. Чем больше оценки вы добавите, тем больше действий по улучшению вы увидите на странице действия по улучшению, а на более высоком — в качестве знаменателя очков.

Чтобы перейти к странице "оценки", перейдите на вкладку **оценки** на панели мониторинга.

На этой странице вы можете быстро просмотреть важную информацию по каждой оценке:

- **Состояние**: состояние завершения всех действий по улучшению оценки будет отображаться следующим образом:
    - Не **соответствует требованиям**: действия по улучшению в оценке не реализованы и успешно протестированы; трудозатраты еще не начались
    - **В ходе выполнения**: действия, выполняемые при реализации или тестировании действий по улучшению; Например, действие по улучшению оценки назначено для работы, находится в процессе реализации и тестирования
- **Ход оценки**: процент работы, выполненной для окончательного завершения оценки, измеряемый числом успешно проверенных элементов управления.
- **Действия, управляемые клиентами**: количество выполненных действий в соответствии с реализацией управляемых пользователем элементов управления
- **Действия, управляемые корпорацией Майкрософт**: количество выполненных действий для реализации управляемых Майкрософт элементов управления
- **Группа оценки**: имя созданной группы, в которой находится Оценка
- **Связанные службы**: связанная служба Microsoft 365
- **Связанные нормативные положения**: нормативный стандарт, политика или закон, которым ищется соответствие при оценке

### <a name="default-assessments"></a>Оценки по умолчанию

По умолчанию на странице оценки будет отображаться Базовая оценка защиты данных Microsoft 365. Оценка соответствия также обеспечивает несколько предварительно настроенных оценок ([Просмотреть полный список](compliance-score.md#templates)). Вы можете добавить дополнительные оценки, чтобы охватить дополнительные нормативные и нормативные стандарты в диспетчере соответствия требованиям.

### <a name="managing-assessments"></a>Управление оценками

Во время общедоступной предварительной версии вы переходите к инструменту "Диспетчер соответствия требованиям", чтобы создавать и настраивать оценки, а также управлять ими.

На странице " **оценки** " показателя соответствия выберите пункт **управление оценкой в диспетчере соответствия требованиям** в верхней части списка оценки. Эта ссылка позволяет перейти на панель мониторинга **оценки** в диспетчере соответствия требованиям.

Вторая ссылка в верхней части списка оценки, **действия Майкрософт в диспетчере соответствия требованиям**позволяет перейти на информационную панель **управления сведениями** в диспетчере соответствия требованиям, в которой будут отображаться элементы управления Майкрософт, которые вносят вклад в оценку соответствия требованиям.

### <a name="filtering-your-assessments-view"></a>Фильтрация представления оценки

Чтобы отфильтровать оценки, выполните следующие действия:

1. Выберите пункт **Фильтр** в левом верхнем углу списка оценки.
2. В области " **Фильтрация** меню" установите флажок рядом с нужным критерием (стандарты и нормативов, Группа диспетчера соответствия требованиям).
3. Нажмите кнопку **Apply (применить** ). Область фильтра будет закрыта, и вы увидите отфильтрованное представление.

Вы также можете изменить представление, чтобы просмотреть оценки по группам, продуктам или нормативным данным, выбрав тип группировки в раскрывающемся меню **Группа** над списком оценки.

### <a name="managing-improvement-actions-within-an-assessment"></a>Управление действиями по улучшению в рамках оценки

В списке Оценка в столбце **действия клиентов — управляемые действия** выберите связанный текст в строке предполагаемой оценки. В результате вы переводите отфильтрованное представление страницы действий по улучшению, в которой отображаются действия, выполняемые в рамках этой оценки.

## <a name="reporting"></a>Reporting

Вы можете экспортировать отчет обо всех действиях по улучшению в рейтинге соответствия требованиям. На странице **действия по улучшению** выберите пункт **Экспорт** в левом верхнем углу экрана над списком действий. Это приведет к созданию листа Excel со всеми действиями по улучшению и категориями фильтров, которые отображаются на странице **действия по улучшению** .

Кроме того, вы можете экспортировать отчет из диспетчера соответствия требованиям, выполнив указанные ниже действия.

1. В диспетчере соответствия требованиям перейдите на панель мониторинга " **сведения об элементах управления** ".
2. Вы увидите вкладку **Оценка** и **шаблон** .  
3. Чтобы экспортировать оценку, выберите вкладку **Оценка** . Выберите раскрывающееся меню **Group** and **оценок** , чтобы выбрать оценку, которую нужно экспортировать. Выберите пункт **Экспорт** в правом верхнем углу экрана. Будет загружен файл Excel. Он включает список действий, сгруппированных по контролю, с описанием их реализации и тестирования.
4. Чтобы экспортировать шаблон, выберите вкладку **шаблон** и выберите шаблон, который нужно экспортировать, в раскрывающемся меню **шаблон** . Выберите пункт **Экспорт** в правом верхнем углу экрана. Будет загружен файл Excel. Он включает список действий, сгруппированных по контролю, с описанием их реализации и тестирования.
