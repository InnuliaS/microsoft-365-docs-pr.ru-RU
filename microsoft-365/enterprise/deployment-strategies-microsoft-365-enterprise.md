# <a name="microsoft-365-enterprise-foundation-infrastructure-deployment-strategies"></a>Стратегии развертывания базовой инфраструктуры Microsoft 365 корпоративный

Существует множество способов, с помощью которых можно развернуть этапы [базовой инфраструктуры](deploy-foundation-infrastructure.md) Microsoft 365 корпоративный и реализовать ее возможности, программное обеспечение и службы для пользователей. Чтобы начать управление таким проектом, который может быть масштабным и сложным в зависимости от размера организации и существующей инфраструктуры, рассмотрите указанные ниже стратегии развертывания.

- Последовательное развертывание
- Параллельное развертывание без перекрытия пользовательского развертывания
- Параллельное развертывание с перекрытием пользовательского развертывания
- Предварительная подготовка инфраструктуры и внедрение комплексной настройки

Используйте эти стратегии для выбора способа управления общим проектом и ускоренной реализации бизнес-преимуществ Microsoft 365 корпоративный.

>[!Note]
>В этой статье содержатся допущения и упрощения для единообразного способа описания стратегий развертывания. Эти стратегии развертывания являются обобщенными, не подразумевают конкретных временных рамок и не предназначены для применения во всех организациях и ситуациях.
>

## <a name="elements-of-it-project-management-for-typical-enterprise-organizations"></a>Элементы управления ИТ-проектами для типовых корпоративных организаций

ИТ-инфраструктура включает серверные службы, развертывание новых или улучшенных возможностей или установленного программного обеспечения для конечных пользователей. Развертывание ИТ-отделами элементов ИТ-инфраструктуры обычно выполняется систематично. Этапы одного из способов успешного развертывания элемента ИТ-инфраструктуры:

- Пилотное развертывание 

  Включает исходную настройку инфраструктуры и развертывание для пилотной группы пользователей, тестирование и последующие изменения в настройке инфраструктуры.

- Пользовательское развертывание

  Включает развертывание для остальной организации на основе областей, отделов, групп и других типов системного распространения настройки или программного обеспечения.

Группа пользователей в пилотном развертывании отличается от группы пользовательского развертывания.

В этой статье используются указанные ниже изображения для этих определений. 

![](./media/deployment-strategies-microsoft-365-enterprise/definitions.png) 

Затенение изображения пользовательского развертывания указывает процентное значение в организации от 0 до 100 % с использованием структурного или методичного подхода, например по группам, отделам или областям.

## <a name="deployment-strategies"></a>Стратегии развертывания

Рассмотрите указанные ниже стратегии развертывания.

- Последовательное развертывание
- Параллельное развертывание без перекрытия пользовательского развертывания
- Параллельное развертывание с перекрытием пользовательского развертывания
- Предварительная подготовка инфраструктуры и внедрение комплексной настройки

### <a name="serial-deployment"></a>Последовательное развертывание

При последовательном развертывании перед переходом к следующему этапу выполняется полное развертывание текущего этапа с достижением 100 % завершения для всех пользователей. Причины возможного применения этого способа развертывания:

- Устранение рисков
- Ограничения ресурсов
- Циклы финансирования ИТ-отделов
- Зависимости от ИТ-технологии
- Управление изменениями в компании и сопротивление конечных пользователей

На этой диаграмме Ганта показано упрощенное последовательное развертывание этапов 2–6 базовой инфраструктуры для Microsoft 365 корпоративный.

![](./media/deployment-strategies-microsoft-365-enterprise/serial.png) 
 
Для упрощения обсуждения и примера предполагается, что каждый этап и сегмент развертывания в каждом этапе занимают одинаковое время.

>[!Note]
>Этап 1. Сетевые подключения базовой инфраструктуры Microsoft 365 корпоративный являются этапом, относящимся только к ИТ-отделу. Пользователи используют преимущества оптимизированного подключения к облачным ресурсам Майкрософт, но не обязаны участвовать в реализации этого этапа.
>

Упрощенный пример опыта пилотного пользователя:

- В декабре пользователю нужно использовать свой смартфон для многофакторной проверки подлинности. (Удостоверение)
- В марте на компьютере пользователя с Windows 8.1 устанавливается Windows 10 Корпоративная. (Windows 10 Корпоративная)
- В июне для пользователя устанавливается Office 365 профессиональный плюс, заменяющий Office 2013. (Office 365 профессиональный плюс)
- В сентябре выполняется развертывание устройства и применение политик приложения и условного доступа. (Управление мобильными устройствами)
- В декабре для пользователя устанавливается клиент Azure Information Protection и он проходит обучение по применению меток к документам. (Защита информации)

В результате обеспечивается периодичность в 90 дней между успешными пилотными развертываниями.

Упрощенный пример опыта конечного пользователя:

- В январе пользователю нужно использовать свой смартфон для многофакторной проверки подлинности. (Удостоверение)
- В апреле на компьютере пользователя с Windows 8.1 устанавливается Windows 10 Корпоративная. (Windows 10 Корпоративная)
- В июле для пользователя устанавливается Office 365 профессиональный плюс, заменяющий Office 2013. (Office 365 профессиональный плюс)
- В октябре выполняется развертывание устройства и применение политик приложения и условного доступа. (Управление мобильными устройствами)
- В январе следующего года для пользователя устанавливается клиент Azure Information Protection и он проходит обучение по применению меток к документам. (Защита информации)

В результате обеспечивается периодичность в 90 дней между успешными пользовательскими развертываниями.

Недостатком этой стратегии развертывания является возможная продолжительность полного развертывания базовой инфраструктуры Microsoft 365 корпоративный.

### <a name="parallel-deployment-with-non-overlapping-user-rollout-parallel-1"></a>Параллельное развертывание без перекрытия пользовательского развертывания (параллельное 1)

При этой стратегии пилотное развертывание следующего этапа начинается на последнем шаге пользовательского развертывания текущего этапа. Ниже представлено развертывание этапов 2–6 при выполнении пилотного развертывания во время завершения пользовательского развертывания предыдущего этапа.

Ниже приведено упрощенное сравнение стратегий параллельного и последовательного развертывания.

![](./media/deployment-strategies-microsoft-365-enterprise/parallel1.png) 
 
Итоговым результатом является завершение в организации пользовательского развертывания текущего этапа до начала следующего. Пользователи, не участвующие в пилотных развертываниях, не сталкиваются с одновременным развертыванием нескольких этапов, но пилотные развертывания выполняются параллельно с пользовательскими.

Упрощенный пример опыта пилотного пользователя: 

- В декабре пользователю нужно использовать свой смартфон для многофакторной проверки подлинности. (Удостоверение)
- В феврале на компьютере пользователя с Windows 8.1 устанавливается Windows 10 Корпоративная. (Windows 10 Корпоративная)
- В апреле для пользователя устанавливается Office 365 профессиональный плюс, заменяющий Office 2013. (Office 365 профессиональный плюс)
- В июне выполняется развертывание устройства и применение политик приложения и условного доступа. (Управление мобильными устройствами)
- В августе для пользователя устанавливается клиент Azure Information Protection и он проходит обучение по применению меток к документам. (Защита информации)

В результате обеспечивается периодичность в 60 дней между успешными пилотными развертываниями.

Упрощенный пример опыта конечного пользователя:

- В январе пользователю нужно использовать свой смартфон для многофакторной проверки подлинности. (Удостоверение)
- В марте на компьютере пользователя с Windows 8.1 устанавливается Windows 10 Корпоративная. (Windows 10 Корпоративная)
- В мае для пользователя устанавливается Office 365 профессиональный плюс, заменяющий Office 2013. (Office 365 профессиональный плюс)
- В июле выполняется развертывание устройства и применение политик приложения и условного доступа. (Управление мобильными устройствами)
- В сентябре для пользователя устанавливается клиент Azure Information Protection и он проходит обучение по применению меток к документам. (Защита информации)

В результате обеспечивается периодичность в 60 дней между успешными пользовательскими развертываниями.

Преимущество этой стратегии развертывания заключается в возможности быстрее выполнить полное развертывание базовой инфраструктуры Microsoft 365 корпоративный без необходимости ИТ-отделу и пользователям сталкиваться с несколькими развертываниями одновременно.

### <a name="parallel-deployment-with-overlapping-user-rollout-parallel-2"></a>Параллельное развертывание с перекрытием пользовательского развертывания (параллельное 2)

При этой стратегия развертывания вы начинаете:

- пилотное развертывание следующего этапа во время последней части пользовательского развертывания текущего этапа.
- пользовательское развертывание следующего этапа во время пользовательского развертывания текущего этапа таким образом, что пользователям не приходится сталкиваться одновременным развертыванием нескольких этапов. Предполагается, что развертывание каждого этапа базовой инфраструктуры выполняется аналогично (по областям, отделам или другим единицам).

Ниже приведено упрощенное сравнение разных стратегий развертывания.

![](./media/deployment-strategies-microsoft-365-enterprise/parallel2.png) 

Итоговый результат:

- Переход пилотных развертываний с одного этапа на следующий выполняется без задержек.
- Пользовательское развертывание для этапа начинается до завершения пользовательского развертывания предыдущего этапа, но отдельные пользователи не выполняют одновременное развертывание нескольких этапов.

Упрощенный пример опыта пилотного пользователя: 

- В декабре пользователю нужно использовать свой смартфон для многофакторной проверки подлинности. (Удостоверение)
- В январе на компьютере пользователя с Windows 8.1 устанавливается Windows 10 Корпоративная. (Windows 10 Корпоративная)
- В феврале для пользователя устанавливается Office 365 профессиональный плюс, заменяющий Office 2013. (Office 365 профессиональный плюс)
- В марте выполняется развертывание устройства и применение политик приложения и условного доступа. (Управление мобильными устройствами)
- В апреле для пользователя устанавливается клиент Azure Information Protection и он проходит обучение по применению меток к документам. (Защита информации)

В результате обеспечивается периодичность в 30 дней между успешными пилотными развертываниями.

Упрощенный пример опыта конечного пользователя:

- В январе пользователю нужно использовать свой смартфон для многофакторной проверки подлинности. (Удостоверение)
- В феврале на компьютере пользователя с Windows 8.1 устанавливается Windows 10 Корпоративная. (Windows 10 Корпоративная)
- В марте для пользователя устанавливается Office 365 профессиональный плюс, заменяющий Office 2013. (Office 365 профессиональный плюс)
- В апреле выполняется развертывание устройства и применение политик приложения и условного доступа. (Управление мобильными устройствами)
- В мае для пользователя устанавливается клиент Azure Information Protection и он проходит обучение по применению меток к документам. (Защита информации)

В результате обеспечивается периодичность в 30 дней между успешными пользовательскими развертываниями.

Преимущество этой стратегии развертывания заключается в возможности еще быстрее выполнить полное развертывание базовой инфраструктуры Microsoft 365 корпоративный без необходимости отдельным пользователям сталкиваться с несколькими развертываниями одновременно. Но для пользователей отсутствует перерыв между успешно выполненными этапами.

### <a name="up-front-infrastructure-and-rollout-of-end-to-end-configuration"></a>Предварительная настройка инфраструктуры и внедрение комплексной настройки

Для небольших организаций с возможностью сжатия этапов 2–6 в один сегмент развертывания итоговое развертывание выглядит представленным ниже образом.
 
![](./media/deployment-strategies-microsoft-365-enterprise/up-front.png) 

ИТ-отдел настраивает инфраструктуру для этапов 2–6, а затем развертывает ее для пилотных пользователей, чтобы проверить комплексные функции. Например, пилотные пользователи получают все эти функциональные возможности одновременно:

- многофакторная проверка подлинности и другие функции удостоверений (удостоверение);
- Windows 10 Корпоративная на устройствах с Windows (Windows 10 Корпоративная);
- Office 365 профессиональный плюс для набора Office (Office 365 профессиональный плюс);
- политики приложения и условного доступа (управление мобильными устройствами);
- установленный клиент Azure Information Protection и обучение по применению меток к документам (защита информации).

После завершения пилотного развертывания начинается пользовательское развертывание, при котором пользователь получает все функциональные возможности одновременно.
