<span data-ttu-id="8155b-107">Azure AD Connect — это поддерживаемое корпорацией Майкрософт средство, с помощью которого вы сможете синхронизировать только те удостоверения, которые вам действительно нужны, из сред AD DS с одним или несколькими лесами в свой клиент Azure AD.</span><span class="sxs-lookup"><span data-stu-id="8155b-107">Azure AD Connect is the supported Microsoft tool that guides you through synchronizing only the identities you really need from single or multi-forest AD DS environments to your Azure AD tenant.</span></span> <span data-ttu-id="8155b-108">На рисунке ниже показан базовый процесс для синхронизации Azure AD Connect.</span><span class="sxs-lookup"><span data-stu-id="8155b-108">The following figure shows the basic process for Azure AD Connect synchronization.</span></span>

1. <span data-ttu-id="8155b-110">Средство Azure AD Connect, запущенное на сервере, опрашивает службы AD DS на наличие изменений в учетных записях, группах и контактах.</span><span class="sxs-lookup"><span data-stu-id="8155b-110">Azure AD Connect running on a server polls AD DS for changes in accounts, groups, and contacts.</span></span>

<span data-ttu-id="8155b-p102">Первое решение, которое необходимо принять касательно гибридного решения для работы с удостоверениями, связано с требованием проверки подлинности. - <span data-ttu-id="8155b-p103">С помощью **управляемой проверки подлинности** Azure AD выполняет процесс проверки подлинности при входе пользователей в систему. There are two methods for managed authentication:</span></span> 
    - <span data-ttu-id="8155b-116">**Синхронизация хэша паролей (PHS)**. [Рекомендованный и обязательный метод для некоторых премиум-функций.]</span><span class="sxs-lookup"><span data-stu-id="8155b-116">**Password Hash Sync (PHS)** [Recommended and required for some premium features].</span></span> <span data-ttu-id="8155b-117">Это самый простой способ включения проверки подлинности для объектов локального каталога в Azure AD.</span><span class="sxs-lookup"><span data-stu-id="8155b-117">This is the simplest way to enable authentication for on-premises directory objects in Azure AD.</span></span> <span data-ttu-id="8155b-118">Azure AD Connect извлекает хэшированный пароль из AD DS, выполняет дополнительную обработку пароля для повышения безопасности и сохраняет его в Azure AD.</span><span class="sxs-lookup"><span data-stu-id="8155b-118">Azure AD Connect extracts the hashed password from AD DS, does extra security processing on the password, and saves it in Azure AD.</span></span> <span data-ttu-id="8155b-119">Дополнительные сведения см. в статье о [реализации синхронизации хэша паролей с помощью службы синхронизации Azure AD Connect](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnectsync-implement-password-synchronization).</span><span class="sxs-lookup"><span data-stu-id="8155b-119">For more information, see [Implement password hash synchronization with Azure AD Connect sync](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnectsync-implement-password-synchronization).</span></span>
    - <span data-ttu-id="8155b-120">**Сквозная проверка подлинности** — это простое решение для проверки подлинности паролей для служб на основе Azure AD.</span><span class="sxs-lookup"><span data-stu-id="8155b-120">**Pass-through Authentication (PTA)** provides a simple password validation solution for Azure AD-based services.</span></span> <span data-ttu-id="8155b-121">При сквозной проверке подлинности используется агент, работающий на одном или нескольких локальных серверах и проверяющий операции проверки подлинности пользователей непосредственно с помощью локальных служб AD DS.</span><span class="sxs-lookup"><span data-stu-id="8155b-121">PTA uses an agent running on one or more on-premises servers to validate the user authentications directly with your on-premises AD DS.</span></span> <span data-ttu-id="8155b-122">Дополнительные сведения см. в статье [Вход пользователей с помощью сквозной проверки подлинности Azure Active Directory](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication).</span><span class="sxs-lookup"><span data-stu-id="8155b-122">For more information, see [User sign-in with Azure Active Directory Pass-through Authentication](https://docs.microsoft.com/azure/active-directory/connect/active-directory-aadconnect-pass-through-authentication).</span></span>
- <span data-ttu-id="8155b-p106">При использовании **федеративной проверки подлинности** процесс проверки подлинности перенаправляется другому поставщику удостоверений через сервер федерации удостоверений, например через службы федерации Active Directory (AD FS), для входа пользователя в систему.
