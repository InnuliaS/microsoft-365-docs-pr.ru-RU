# <a name="phase-5-mobile-device-management-for-microsoft-365-enterprise"></a>Этап 5: Управление мобильными устройствами для Microsoft 365 корпоративный

![](./media/deploy-foundation-infrastructure/mobiledevicemgmt_icon.png)

*Эта функция применяется к E3 и E5 версий Microsoft 365 для предприятия*

Microsoft 365 Enterprise включает в себя функции для управления устройствами и их приложения в вашей организации. С помощью Microsoft Intune, можно управлять операций ввода-вывода, Android, macOS и устройств Windows для защиты доступа к ресурсам вашей организации, включая данные. Intune интегрируется с Azure Active Directory (Azure AD) и включает следующие бизнес-сценарии для Microsoft 365:

- возможность хранить файлы и делиться ими в организации и за ее пределами для обеспечения эффективности работы;
- безопасная работа на любых устройствах где угодно и когда угодно для обеспечения эффективности при гибком стиле работы;
- беспроблемное применение средств контроля и реализация прозрачности согласно глобальным стандартам обеспечения соответствия требованиям;
- защита данных и снижение риска утери данных;
- Обнаружение и обеспечивает защиту от внешних угроз
- Мониторинг, создавать отчеты и анализировать действия быстро реагировать для обеспечения безопасности предприятия
- Защиты пользователей и учетных записей

Дополнительные сведения см. в статье [Цифровая трансформация с помощью Microsoft 365](http://transform.microsoft.com). 

На этом этапе Регистрация устройств в Intune и создания и применения политик для защиты данных и защитить. Всю библиотеку Intune документация является [Доступные online](https://docs.microsoft.com/intune). Рекомендуется также просмотр [Intune развертывания планирование, проектирование и реализация руководство](https://docs.microsoft.com/intune/planning-guide) перед началом работы.

## <a name="step-1-plan-for-your-scenario"></a>Этап 1: Планирование для сценария

Одной из основных причин для управления мобильных устройств является для обеспечения безопасности и защиты ресурсов вашей организации. [Распространенные способы использования Microsoft Intune](https://docs.microsoft.com/intune/common-scenarios) приведены некоторые примеры, включая защиты электронной почты Office 365 и данных.

Intune предоставляет параметры для управления доступом к вашей организации, с помощью [мобильных устройств управления (MDM) или мобильного приложения управления (MAM)](https://docs.microsoft.com/intune/byod-technology-decisions). MDM — когда пользователи «регистрация» свои устройства Intune. После регистрации, они являются управляемыми устройствами и может получить все политики, правила и параметры, используемые вашей организации. Например можно устанавливать приложения особенности, создание политики паролей, установить VPN-подключение и многое другое.

Пользователи, имеющие собственные личные устройства может потребоваться зарегистрировать свои устройства или управляться Intune и политиках. Но по-прежнему требуется для защиты ресурсов и данных вашей организации. В этом сценарии можно защитить с помощью MAM приложений. Например можно использовать MAM политики, которая требуется для ввода ПИН-код при доступе к SharePoint на устройстве.

Необходимо также решить, как вы собираетесь управлять личных или владении организации устройств. Можно рассматривать устройств по-разному в зависимости от их использования. Например можно различных планов для пользователей в отдел кадров (HR) или пользователи объема продаж. [Определение мобильных устройств management сценариев](https://docs.microsoft.com/intune/planning-guide-scenarios) можно приступить к работе и включает в себя некоторые рекомендации на этих сценариев.

## <a name="step-2-get-your-prerequisites"></a>Шаг 2: Получение предварительных требований

Затем получите предварительных требований на основе требований и своих сценариев, созданных на предыдущем шаге. [Реализация плана](https://docs.microsoft.com/intune/planning-guide-onboarding) перечисляются все требования. Вот значительные элементы, которые необходимы для Intune 365 Microsoft.

- **Подписки Intune**: входит в состав Microsoft 365 и предоставляет доступ к Intune Microsoft [Azure портала](https://portal.azure.com)
- **Подписки на Office 365**: входит в состав Microsoft 365 и используется для приложений Office, включая электронной почты
- **Расширенный Azure Active Directory (AD)**: входит в состав Microsoft 365 и используется для создания пользователей или групп безопасности. Эти группы получают Intune политики, созданные вами, таких как принудительным длина пароля для разблокировки устройства. Группы, создайте в [Этап 2: удостоверение](https://docs.microsoft.com/microsoft-365/enterprise/identity-infrastructure) можно использовать.

Возможно, некоторые дополнительные требования, в зависимости от потребностей организации. Например будет управления iOS устройств, вам потребуются сертификат Apple MDM Push. Если локальный сервер Exchange, то вам потребуются локальный соединитель Exchange. Если все эти шаги описаны следующие дополнительные требования.

## <a name="step-3-set-up-intune"></a>Шаг 3: Настройка Intune

Intune использует многие функции в Azure AD, включая ваш домен, пользователей и групп. Можно также создать новых пользователей и новые группы в соответствии со своими потребностями организации. Например можно создать группу с именем **устройства iOS**или **отдела Кадров всех пользователей**.  Использование [Динамических групп](https://docs.microsoft.com/intune/groups-add) , позволяющий создавать пользователей или групп устройств на основе вокруг простых или расширенных правил.

Это действие основное внимание уделяется Настройка Intune и подготовка управления устройствами.

1. **[Подтверждение, поддерживаемые устройства](https://docs.microsoft.com/intune/supported-devices-browsers)**. Подтверждение iOS, macOS, поддерживаемых устройства Android, галактики и Windows Intune. Если в организации имеется устройств, которые не поддерживаются, политики не применяется для этих устройств.

2. **[Настройка имени домена](https://docs.microsoft.com/intune/custom-domain-name-configure)**. По умолчанию домен с именем вроде **your domain.onmicrosoft.com** автоматически создается в Azure AD. **onmicrosoft.com** можно настроить для вашей организации. При настройке, он также предоставляет пользователям привычный домена при подключении к Intune и использование ресурсов.

3. **[Вход на портал Intune](https://docs.microsoft.com/intune/account-sign-up)**. При входе в может потребоваться ввод сведений о вашей организации. Intune входит в состав Microsoft 365 и его можно открыть прямо с [портала администрирования Office 365](https://portal.office.com/). Можно также открыть Intune прямо с [портала Azure](https://portal.azure.com).

4. **[Выберите конфигурацию управления мобильного устройства](https://docs.microsoft.com/intune/mdm-authority-set)**. При первом использовании Intune, необходимо включить управление устройствами. Intune можно использовать в качестве только в облаке службы гибридного с Intune и System Center Configuration Manager или с помощью мобильных устройств Management for Office 365. Можно выбрать, какие программы установки из них лучше подходит для вашей организации.

5. **[Добавление пользователей](https://docs.microsoft.com/intune/users-add)** и **[добавлять группы](https://docs.microsoft.com/intune/groups-add)**. 

   Можно вручную добавить пользователей или подключитесь к Azure AD для синхронизации пользователей с Intune. Можно также предоставить роли администраторов к определенным пользователям. Пользователи не требуется, если устройств, которые «userless» устройств, таких как киоска.

   Azure AD группы используются для упрощения управления устройствами и пользователями в Intune. Использование групп, это можно сделать решения различных задач. Например организации необходимо требуют определенного приложения на устройствах Android. Можно создать группу устройства Android и развертывания политики с помощью этого приложения в группу.

    В Intune, можно добавить пользователей или группы, создаваемые в [Этап 2: удостоверение](https://docs.microsoft.com/microsoft-365/enterprise/identity-infrastructure)

6. **[Назначение лицензий](https://docs.microsoft.com/intune/licenses-assign)**. Для пользователей или устройств для регистрации в Intune они должны лицензии на устройстве. Userless устройства или каждому пользователю требуется лицензия Intune для доступа к службе Intune. Эти лицензии входящих в состав Microsoft 365 и должны быть назначены Intune.

## <a name="step-4-enroll-devices"></a>Шаг 4: Регистрация устройств

Или можно автоматизировать регистрации так просто входе пользователей в устройстве.

При использовании Intune регистрации является важным шагом. [Регистрация устройств](https://docs.microsoft.com/intune/device-enrollment) приведены шаги для различных устройств.

## <a name="step-6-turn-on-compliance-and-conditional-access"></a>Шаг 6: Включение контроля соответствия требованиям и условного доступа

На предыдущих шагах настройки среды и поддержкой Intune. Теперь вы готовы создать несколько политик, с помощью контроля соответствия требованиям и условного доступа.

Соответствие требованиям и условного доступа важны для управления устройствами. **[Политики соответствия](https://docs.microsoft.com/intune/device-compliance-get-started)** создаются для защиты ресурсов вашей организации. При создании политики соответствия требованиям, заданных standard или «базовой» из устройства, которые должны иметь. К примеру можно выбирать угрозы приемлемой (или недопустимое), блокировки устройств jailbroken, требуют длина пароля и многое другое. Если эти устройства не отвечают правил, что означает, что они не соответствует спецификации, можно заблокировать доступ к ресурсам.

Это «блокировка» представляется **[условного доступа](https://docs.microsoft.com/intune/conditional-access)**. Если устройство считается не соответствует спецификации, можно заблокировать доступ к электронной почте, SharePoint и др.

Intune [Azure портала](https://portal.azure.com) позволяет создавать эти политики и применить их для пользователей и устройств. Рекомендуется запустите небольшой и использовать поэтапный подход. Например создание политики операций ввода-вывода, который блокирует jailbroken устройств. Применение политики (называемое «назначение» в Intune) в группу пилотной или тестирования. После первоначальной проверки добавьте большее количество пользователей пробной группы. С помощью поэтапной подход, вы можете получить свои отзывы и предложения на широкий набор типов пользователей.

[Начало работы с политики соответствия устройства](https://docs.microsoft.com/intune/device-compliance-get-started) и [возможности условного доступа?](https://docs.microsoft.com/intune/conditional-access) , которые помогут приступить к работе.

## <a name="step-7-apply-features-and-settings"></a>Шаг 7: Применение функции и параметры

Эти компоненты и параметры часто считаются «интересные» частью Intune и очень полезны. После успешного принудительном некоторые политики соответствия требованиям, с помощью условного доступа вы готовы для создания **профилей устройств**.

Intune [Azure портала](https://portal.azure.com) позволяет создавать различные профили на основе платформы устройства - операций ввода-вывода, macOS, Android и Windows. Например можно выполнить следующие действия.

- Используйте защиты конечной точки на устройствах Windows 10 для включения различные варианты BitLocker, включая шифрования.
- Используйте функцию ограниченных приложений на устройствах iOS создает список одобренных приложений, которые можно установить. Или создать список запрещенных приложений.
- Используйте параметры киоска выбирать, какие приложения можно использовать на устройствах Android, выполняющегося в полноэкранном режиме.
- Применение подключения Wi-Fi и его параметры, в том числе тип безопасности на устройствах под управлением macOS.
- И др.

[Каковы профилей устройств Microsoft Intune?](https://docs.microsoft.com/intune/device-profiles) — Это прекрасная возможность ознакомьтесь со сведениями о профилях, узнайте, как создать профиль и многое другое.

Не забывайте, запустите небольшой и использовать поэтапный подход. Назначьте профиль группе пилотной или тестирования. Назначьте профиль для нескольких групп пилотного проекта.
