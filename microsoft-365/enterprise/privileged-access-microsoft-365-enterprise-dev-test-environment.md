# <a name="privileged-access-management-for-your-microsoft-365-enterprise-test-environment"></a>Управление привилегированным доступом для вашей тестовой среды Microsoft 365 корпоративный

Инструкции в этом разделе настроить управление правами доступа для повышения безопасности в тестовой среде Microsoft 365 Enterprise.

![Руководства по лаборатории тестирования для Microsoft Cloud](media/m365-enterprise-test-lab-guides/cloud-tlg-icon.png)

> [!TIP]
> Щелкните [здесь](https://aka.ms/m365etlgstack), чтобы просмотреть схему всех статей, относящихся к руководствам по лаборатории тестирования Microsoft 365 корпоративный.
  
## <a name="phase-1-build-out-your-microsoft-365-enterprise-test-environment"></a>Этап 1: Создание масштабирование тестовой среды Microsoft 365 для предприятия

Если необходимо настроить управление правами доступа в облегченный путь с минимальным требованиям, следуйте инструкциям в [упрощенной базовой конфигурации](lightweight-base-configuration-microsoft-365-enterprise.md).
  
Если вы хотите настроить управление правами доступа имитации enterprise, следуйте инструкциям в [сквозная проверка подлинности](pass-through-auth-m365-ent-test-environment.md).
  
> [!NOTE]
> Тестирование управления правами доступа не требуется тестовой среды имитации предприятия, включающая имитации интрасети, подключенных к Интернету и синхронизации каталогов для леса Windows Server AD. Здесь предоставляются как вариант таким образом, вы можете проверить доступ к управлению правами доступа и экспериментировать с его в среде, которая представляет типичное организации. 

## <a name="phase-2-configure-privileged-access-management"></a>Этап 2: Настройка управления правами доступа

На этом этапе настройки группы обеспечения утверждающие и включить управление правами доступа для тестовой среды Microsoft 365 для предприятия. Для дополнительных сведений о и общие сведения о полномочиями доступ к управлению, см [Управление правами доступа в Office 365](https://docs.microsoft.com/office365/securitycompliance/privileged-access-management-overview).

Выполните следующие действия для установки и использования привилегированной доступа в организации Office 365.

- [Шаг 1: Создание группы утверждающий](https://docs.microsoft.com/office365/securitycompliance/privileged-access-management-configuration#step-1---create-an-approvers-group)

    Прежде чем начать, с помощью принципу предоставления минимальных прав доступа, определение пользователей, которые будут иметь Центр утверждения для входящих запросов для доступа к задачам с повышенными привилегиями и правами. Любой пользователь, который является частью в группе утверждающих будет иметь возможность утверждать запросы доступа. Этот параметр включен, создав группу безопасности с включенной поддержкой почты в Office 365. Создать новую группу безопасности с именем «Привилегированной утверждающие доступа» в тестовой среде и добавьте «пользователя 3" ранее созданные в предыдущих шагов руководство по лаборатории.

- [Шаг 2: Включение привилегированный доступ](https://docs.microsoft.com/office365/securitycompliance/privileged-access-management-configuration#step-2---enable-privileged-access)

    Привилегированный доступ должен явно включены в Office 365 с помощью группы утверждающий по умолчанию и включая набор системных учетных записей, которые необходимо исключить из управления доступом управления правами доступа. Не забудьте включить привилегированный доступ в организации Office 365, прежде чем начинать этап 3 в этом руководстве.

## <a name="phase-3-verify-that-approval-is-required-for-elevated-and-privileged-tasks"></a>Этап 3: Убедитесь, что утверждения, необходимые для выполнения задач с повышенными привилегиями и правами
На этом этапе вы убедитесь, что работа политики привилегированный доступ и пользователям требуется утверждение для выполнения определенных задач с повышенными привилегиями и правами.

### <a name="test-ability-to-execute-a-task-not-defined-in-a-privileged-access-policy"></a>Проверка возможности выполнения задач, не определенных в политике привилегированный доступ

Во-первых подключение к Exchange управления PowerShell с использованием учетных данных пользователя, настроенных как глобальный администратор в тестовой среде и попытаться создать новое правило журнала. [New-JournalRule](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance/new-journalrule?view=exchange-ps) задач в настоящее время не определен в политике привилегированный доступ для вашей организации.

1. Откройте на локальном компьютере и войдите в Exchange Online удаленного модуль PowerShell в **Корпорации Майкрософт** > с помощью глобального администратора**Microsoft Exchange Online PowerShell модуля удаленного** учетных записей для тестовой среды.

2. В Exchange Powershell управления создайте новое правило журнала для вашей организации:

```
New-JournalRule -Name "JournalRule1" -Recipient joe@contoso.onmicrosoft.com -JournalEmailAddress barbara@adatum.com -Scope Global -Enabled $true
```
4. Представления, что новое правило журнала успешно создан в Exchange PowerShell управления.

### <a name="create-a-new-privileged-access-policy-for-the-new-journalrule-task"></a>Создать новую политику привилегированный доступ для задачи New-JournalRule

> [!NOTE]
> Если еще не уже выполнены шаги 1 и 2 из этап 2 в этом руководстве, быть следуя инструкциям для создания группы утверждающий с именем «Утверждающие принципу предоставления минимальных прав доступа» и для включения привилегированный доступ в тестовой среде.

1. Войдите в [Центр администрирования Microsoft 365](https://portal.office.com) , используя учетные данные учетной записи глобального администратора для тестовой среды.

2. В центре администрирования, перейдите в раздел **Параметры** > **безопасности и конфиденциальности** > **правами доступа**.

3. Выберите **Управление политиками доступа и запросов**.

4. Выберите **Настройка политик** , а затем выберите **Добавить политику**.

5. Из раскрывающегося списка полей выберите или введите следующие значения:
    
    **Тип политики**: задачи

    **Область действия политики**: Exchange

    **Имя политики**: новые правила журнала

    **Тип утверждения**: вручную

    **Группа утверждения**: привилегированной утверждающие доступа

6. Выберите **Создать** и затем **Закрыть**. Может занять несколько минут для политики, чтобы полностью настроить и включить. Обязательно Освободите времени для политики, чтобы полностью включить перед началом тестирования требование утверждения на следующем этапе.

### <a name="test-approval-requirement-for-the-new-journalrule-task-defined-in-a-privileged-access-policy"></a>Тестирование требование утверждения для задачи New-JournalRule, определенных в политике привилегированный доступ

1. Откройте на локальном компьютере и войдите в Exchange Online удаленного модуль PowerShell в **Корпорации Майкрософт** > **Microsoft Exchange Online удаленного модуль PowerShell** с помощью глобального администратора в с помощью учетной записи для теста Среда.

2. В Exchange Powershell управления создайте новое правило журнала для вашей организации:

```
New-JournalRule -Name "JournalRule2" -Recipient user1@<your subscription domain> -JournalEmailAddress user1@<your subscription domain> -Scope Global -Enabled $true
```
3. Представление «Insuffient разрешения» ошибка в Exchange PowerShell управления:

```
Insufficient permissions. Please raise an elevated access request for this task.
    + CategoryInfo          : NotSpecified: (:) [], LocalizedException
    + FullyQualifiedErrorId : [Server=CY1PR00MB0220,RequestId=7b8c7470-ddd0-4528-a01e-5e20ecc9bd54,TimeStamp=9/19/2018
    7:38:34 PM] [FailureCategory=Cmdlet-LocalizedException] 882BD051
    + PSComputerName        : outlook.office365.com
```

### <a name="request-access-to-create-a-new-journal-rule-using-the-new-journalrule-task"></a>Запросить доступ для создания нового правила журнала с помощью New-JournalRule задач

1. Войдите в [Центр администрирования Microsoft 365](https://portal.office.com) , используя учетную запись глобального администратора для тестовой среды.

2. В центре администрирования, перейдите в раздел **Параметры** > **безопасности и конфиденциальности** > **правами доступа**.

3. Выберите **Управление политиками доступа и запросов**.

4. Выберите **Создать запрос**. Из раскрывающегося списка полей выберите соответствующие значения для вашей организации:

    **Тип запроса**: задачи

    **Область запроса**: Exchange

    **Запрос**: новые правила журнала

    **Длительность (в часах)**: 2

    **Комментарии**: запрашивать разрешения, чтобы создать новое правило журнала

5. Выберите **Сохранить** и **Закрыть**. Запрос будет отправляться группы утверждающий по электронной почте.

### <a name="approve-privileged-access-request-for-the-creation-of-a-new-journal-rule"></a>Утверждение запроса привилегированный доступ для создания нового правила журнала

1. Войдите в [Центр администрирования Microsoft 365](https://portal.office.com) , используя учетные данные для пользователя 3 в тестовой среде (членство в группе безопасности «Привилегированной утверждающие доступа» в тестовой среде).

2. В центре администрирования, перейдите в раздел **Параметры** > **безопасности и конфиденциальности** > **правами доступа**.

3. Выберите **Управление политиками доступа и запросов**.

4. Выберите ожидающий запрос и **Утвердить** , чтобы предоставить доступ к учетной записи глобального администратора для создания нового правила журнала. По электронной почте уведомление, подтверждающее, предоставления утверждения будет отправлено на учетную запись глобального администратора (запрашивающего пользователя).  

### <a name="test-creating-a-new-journal-rule-with-privileged-access-approved-for-the-new-journalrule-task"></a>Создание нового правила журнала с помощью привилегированной доступа утверждено для задачи New-JournalRule тестирования

1. Откройте на локальном компьютере и войдите в Exchange Online удаленного модуль PowerShell в **Корпорации Майкрософт** > с помощью глобального администратора**Microsoft Exchange Online PowerShell модуля удаленного** учетных записей для тестовой среды.

2. В Exchange Powershell управления создайте новое правило журнала для вашей организации:

```
New-JournalRule -Name "JournalRule2" -Recipient user1@<your subscription domain> -JournalEmailAddress user1@<your subscription domain> -Scope Global -Enabled $true
```
3. Представления, что новое правило журнала успешно создан в Exchange PowerShell управления.
