# <a name="sharepoint-sites-for-highly-regulated-data"></a>Сайты SharePoint для жестко регламентированных данных

*Этот сценарий применяется к планам E3 и E5 Microsoft 365 корпоративный.*

Решение Microsoft 365 корпоративный содержит полный набор облачных служб, предназначенных для создания, хранения и защиты жестко регламентированных данных, хранящихся в файлах, а также для управления ими. К их числу относятся:

- данные, на которые распространяется действие региональных нормативных актов;
- наиболее важные для предприятия данные, например коммерческие тайны, информация о финансовой деятельности или данные о персонале и стратегии организации.

>[!Note]
> Аналогичный сценарий с использованием Microsoft Teams находится в разработке.
>

Облачный сценарий Microsoft 365 корпоративный, который соответствует такой бизнес-потребности предприятия, потребует от вас:

- хранения файлов (документов, презентаций, электронных таблиц и т. д.) на сайте группы SharePoint;
- блокировки сайта, чтобы предотвратить:
  - предоставление доступа пользователям, не являющимся участниками группы Office 365 для этого сайта;
  - предоставление участниками сайта доступа третьим лицам;
  - запросы на доступ к сайту от лиц, которые не являются участниками сайта;
- настройки метки хранения Office 365 для сайтов SharePoint по умолчанию, чтобы блокировать пользователей при отправке файлов за пределы организации;
- шифрования наиболее конфиденциальных файлов сайта таким образом, чтобы файл оставался зашифрованным при перемещении;
- добавления разрешений к наиболее конфиденциальным файлам. Чтобы открыть такие файлы, даже если они будут доступны вне сайта, потребуются действительные учетные данные учетной записи пользователя с соответствующим разрешением. 

В приведенной ниже таблице сопоставляются требования этого сценария и функции Microsoft 365 корпоративный.

|||
|:-------|:-----|
| **Требования** | **Функция Microsoft 365 корпоративный** |
| Хранение файлов | Сайты групп SharePoint |
| Блокировка сайта | Разрешения для групп Azure Active Directory (Azure AD) и сайта группы SharePoint |
| Пометка файлов сайта | Метки хранения Office 365 |
| Блокирование пользователей при отправке файлов за пределы организации | Политики защиты от потери данных в Office 365 |
| Шифрование всех файлов сайта | Вложенные метки конфиденциальности Office 365 |
| Добавление разрешений к файлам сайта | Вложенные метки конфиденциальности Office 365 |
|||

## <a name="phase-1-design"></a>Этап 1. Проектирование

Чтобы создать сайт SharePoint для жестко регламентированных данных, сначала необходимо определить его назначение. Например, отделу исследований и разработок производственного предприятия сайт SharePoint необходим для хранения текущих проектных спецификаций существующих продуктов, а также для совместной работы над новыми продуктами. Доступ к сайту будет разрешен только членам отдела исследований и разработок и избранным руководителям.

Такое назначение сайта потребует определения основных элементов конфигурации, таких как:

- метка хранения Office 365 для назначения разделу сайта "Документы" и набор политик защиты от потери данных для нее;
- параметры вложенной метки конфиденциальности Office 365, которая применяется к высоко конфиденциальным файлам, хранящимся на сайте.

Как только эти параметры будут определены, вы сможете использовать их для настройки сайта на этапе 2. 

### <a name="step-1-office-365-retention-labels-and-dlp-policies"></a>Шаг 1. Метки хранения Office 365 и политики защиты от потери данных

При применении меток хранения Office 365 к разделу "Документы" сайта группы SharePoint указывается заданный по умолчанию метод классификации всех файлов, хранящихся на этом сайте.
 
В отношении сайтов SharePoint для жестко регламентированных данных необходимо определить, какую метку хранения Office 365 следует использовать.

Вопросы проектирования меток Office 365 рассмотрены в статье [Классификация и метки Office 365](https://docs.microsoft.com/office365/securitycompliance/secure-sharepoint-online-sites-and-files#office-365-retention-labels).

Для защиты конфиденциальной информации и предотвращения ее случайного или преднамеренного разглашения используются политики защиты от потери данных. Дополнительные сведения о них см. в этом [обзоре](https://docs.microsoft.com/office365/securitycompliance/data-loss-prevention-policies).

В отношении сайтов SharePoint для жестко регламентированных данных необходимо настроить политику защиты от потери данных для метки хранения Office 365, назначенной такому сайту. Это позволит блокировать пользователей при попытке предоставить доступ к файлам внешним пользователям. 

### <a name="step-2-your-office-365-sensitivity-sublabel"></a>Шаг 2. Вложенная метка конфиденциальности Office 365

Чтобы обеспечить шифрование и набор разрешений для наиболее конфиденциальных файлов, нужно применить вложенную метку конфиденциальности Office 365.

Вложенная метка существует в имеющейся метке. Например, вы можете создать вложенную метку "Исследования и разработки" для метки "Жестко регламентированные данные". Разрешения для сайтов SharePoint для жестко регламентированных данных настраиваются таким образом, чтобы только участники сайта могли открывать и изменять файл, которому присвоена вложенная метка.

Параметры примененной вложенной метки сохраняются при перемещении файла.   Даже в случае утечки за пределы сайта файл можно открыть только с помощью учетных записей пользователей, прошедших проверку подлинности, которые имеют разрешения.


### <a name="design-results"></a>Результаты проектирования

Вы уже определили:

- соответствующую метку хранения Office 365 и политику защиты от потери данных, связанную с этой меткой;
- параметры вложенной метки конфиденциальности Office 365, в том числе шифрование и разрешения.

## <a name="phase-2-configure"></a>Этап 2. Настройка

На этом этапе параметры, определенные на этапе 1, реализуются с целью создания сайта SharePoint для жестко регламентированных данных.

### <a name="step-1-create-a-private-sharepoint-team-site-with-owners-and-members-of-the-corresponding-office-365-group"></a>Шаг 1. Создание частного сайта группы SharePoint владельцами и участниками соответствующей группы Office 365

Чтобы создать частный сайт группы SharePoint, следуйте [инструкциям]( https://support.office.com/article/create-a-site-in-sharepoint-online-4d1e11bf-8ddc-499d-b889-2b48d10b1ce8).

### <a name="step-2-configure-additional-permissions-settings-for-the-sharepoint-team-site"></a>Шаг 2. Настройка параметров дополнительных разрешений для сайта группы SharePoint

Настройте параметры разрешений на сайте SharePoint.

1.  На панели инструментов щелкните значок параметров и выберите вариант **Разрешения для сайта**.
2.  В области **Разрешения для сайта** щелкните **Параметры дополнительных разрешений**.
3.  На новой вкладке браузера **Разрешения** щелкните **Параметры запроса доступа**.
4.  В диалоговом окне **Параметры запросов на доступ** снимите флажки **Разрешить участникам совместный доступ к этому сайту, а также отдельным файлам и папкам** и **Разрешить запросы на доступ** (все три флажка должны быть сняты), а затем нажмите кнопку **OK**.

С помощью этих параметров можно отключить возможность предоставлять общий доступ к сайту другим пользователям для участников группы сайта, а также возможность запрашивать доступ к сайту для лиц, не являющихся участниками.

### <a name="step-3-configure-the-site-for-an-office-365-retention-label"></a>Шаг 3. Настройка сайта для использования метки хранения Office 365

Используйте инструкции, приведенные в статье [Защита файлов SharePoint с помощью меток Office 365 и политики защиты от потери данных](https://docs.microsoft.com/office365/enterprise/protect-sharepoint-online-files-with-office-365-labels-and-dlp), для:

1. создания и публикации метки хранения для жестко регламентированных данных (при необходимости);
2. настройки сайта для использования метки хранения, созданной на шаге 1;
3. создания политики защиты от потери данных для жестко регламентированных данных, которая будет использовать метку хранения, созданную на шаге 2, и блокировать пользователей при отправке файлов за пределы организации.

#### <a name="step-4-create-an-office-365-sensitivity-sublabel-for-the-site"></a>Шаг 4. Создание вложенной метки конфиденциальности Office 365 для сайта

В отличие от метки конфиденциальности для жестко регламентированных данных, которую любой пользователь может применить к любому файлу, защищенный сайт должен иметь собственную вложенную метку, чтобы файлы, которым она назначена:

- были зашифрованы и оставались зашифрованными при перемещении;
-   содержали настраиваемые разрешения, благодаря которым их могут открыть только участники группы сайта.

Чтобы установить этот дополнительный уровень безопасности для файлов, хранящихся на сайте, необходимо настроить новую метку конфиденциальности, являющуюся вложенной меткой для общей метки файлов с жестко регламентированными данными. Только участники группы сайта будут видеть ее в списке вложенных меток для метки жестко регламентированных данных.  

Следуя инструкциям [здесь](https://docs.microsoft.com/microsoft-365/compliance/encryption-sensitivity-labels), настройте вложенную метку для метки, которую вы используете для файлов с жестко регламентированными данными, со следующими параметрами:

- имя вложенной метки должно содержать имя сайта, чтобы легко возникали ассоциации при ее назначении файлу;
- необходимо включить шифрование;
- группа сайта должна иметь разрешения на совместное редактирование.

### <a name="configuration-results"></a>Результаты настройки

Вы настроили:

- более строгие параметры разрешений на сайте SharePoint;
- метку хранения Office 365, назначенную разделу "Документы" сайта SharePoint;
- политику защиты от потери данных для метки хранения Office 365;
- вложенную метку конфиденциальности Office 365, которую пользователи могут применить к наиболее конфиденциальным файлам, хранящимся на сайте. Эта метка шифрует файл и предоставляет доступ для совместного редактирования только участникам сайта группы.

## <a name="phase-3-drive-user-adoption"></a>Этап 3. Внедрение среди пользователей

Сайт SharePoint для жестко регламентированных данных может защищать данные только в том случае, если он постоянно используется для хранения и доступа к конфиденциальным файлам. Это наиболее сложный этап, так как он предполагает изменение привычек и предпочтений пользователей.  

Например, сотрудники, которые ранее хранили конфиденциальные файлы на USB-накопителях или в личных облачных хранилищах, теперь будут вынуждены хранить их только на сайте SharePoint для жестко регламентированных данных.

### <a name="step-1-train-your-users"></a>Шаг 1. Обучение пользователей

После завершения настройки обучите пользователей, входящих в группы доступа к сайту, рассказав им:

- насколько важно использовать новый сайт для защиты ценных файлов, а также о последствиях утечки жестко регламентированных данных, таких как юридические последствия, штрафные санкции за несоблюдение нормативно-правовых актов, появление программ-шантажистов или потеря конкурентного преимущества;
- как получить доступ к сайту и его файлам;
- как создавать файлы на сайте и отправлять новые файлы, которые хранятся локально;
- каким образом политика защиты от потери данных блокирует пользователей при попытке отправить файлы за пределы организации;
- как пометить наиболее конфиденциальные файлы с помощью вложенной метки для сайта;
- как вложенная метка защищает файл даже в случае утечки с сайта.

В программу такого обучения следует включить практические занятия, где пользователи смогут опробовать эти операции и ознакомиться с результатами.

### <a name="step-2-conduct-periodic-reviews-of-usage-and-files"></a>Шаг 2. Проведение периодических проверок использования и файлов

Через несколько недель после обучения администратор SharePoint может выполнить следующие действия в отношении сайта SharePoint:

- проанализировать использование сайта и сравнить результаты с требованиями к использованию;
- убедиться, что наиболее конфиденциальные файлы были надлежащим образом помечены с помощью вложенной метки конфиденциальности.

При необходимости следует провести повторное обучение пользователей.

### <a name="user-adoption-results"></a>Результаты адаптации пользователей к новым требованиям

Файлы с жестко регламентированными данными хранятся только на сайтах SharePoint для жестко регламентированных данных, а к наиболее конфиденциальным файлам применяется вложенная метка конфиденциальности для сайта.
