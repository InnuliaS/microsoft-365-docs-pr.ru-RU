# <a name="step-3-deploy-windows-10-enterprise-for-new-devices-with-windows-autopilot"></a>Шаг 3: развертывание Windows 10 Корпоративная для новых устройств с помощью Windows автопилота

*Эта статья относится как к корпоративным, так и к версиям Microsoft 365 для Microsoft*

![Этап 3. Windows 10 Корпоративная](./media/deploy-foundation-infrastructure/win10enterprise_icon-small.png)

Если у вас есть новые компьютеры с Windows 10, вы можете использовать автопилот Windows для настройки готового к работе (OOBE) для вашей организации и развертывания новой системы с приложениями и настройками, которые уже настроены. Вам не придется развертывать образы, устанавливать драйверы и управлять инфраструктурой. Пользователи могут проходить через процесс развертывания независимо, без необходимости обращаться к ИТ ИТ ИТ.

Вы можете настроить и предварительно настроить новые устройства с Windows 10 и подготовить их к эффективному использованию с помощью Windows "Автопилот". Для получения дополнительных сведений о Windows автопилоте, в том числе о преимуществах и сценариях Windows для автопилота, ознакомьтесь со статьей [Обзор Windows автопилот](https://docs.microsoft.com/windows/deployment/windows-Autopilot/windows-10-Autopilot). Когда вы будете готовы, выполните следующие части, чтобы начать настройку новых устройств.

## <a name="part-1-start-windows-autopilot-deployment"></a>Часть 1: запуск развертывания Windows для автопилота
Просмотрите [Обзор Windows Автопилот для выполнения](https://docs.microsoft.com/windows/deployment/windows-Autopilot/windows-10-Autopilot) следующих действий:

1. Узнайте о предварительных требованиях для развертывания Windows с помощью средства "автопилотная версия". Ниже приведены необходимые условия.
    - **регистрацию устройств и настройку запуска при первом включении компьютера;**

        Для регистрации устройств необходимо получить идентификатор оборудования и зарегистрировать его. Мы активно работаем с различными поставщиками оборудования, чтобы позволить им предоставить вам необходимые сведения или отправить их от вашего имени. Вы также можете записать эти сведения самостоятельно, используя скрипт PowerShell, который создает CSV-файл с аппаратным ИДЕНТИФИКАТОРом устройства.

        После регистрации устройств можно настроить параметры настройки OOBE, включая пропуск параметров конфиденциальности и лицензионного соглашения.

    - **фирменную символику для запуска при первом включении компьютера;**

        Это позволяет добавить фирменную символику, которая будет отображаться во время OOBE устройства.

    - **автоматическую регистрацию MDM в Microsoft Intune;**
        
        Автоматическая регистрация позволяет пользователям регистрировать свои устройства с Windows 10 в Intune для управления устройствами при подключении своих устройств к Azure AD. Чтобы зарегистрировать, пользователи добавляют свои рабочие учетные записи в собственные устройства или присоединяются к устройствам, принадлежащим корпоративным корпоративным устройствам, в Azure AD. В фоновом режиме устройство также регистрируется для управления с помощью Intune.

    - **сетевое подключение к облачным службам, которые использует Windows Autopilot;**

        Программа развертывания Windows для автопилота использует несколько облачных служб для достижения производительного состояния устройств, и эти службы должны быть доступны на устройствах, зарегистрированных как устройства с Windows автопилотной версии. 

    - **предварительную установку Windows 10 версии 1703 или выше на устройствах;**

2. Узнайте о программе развертывания Windows и выберите ее для своей организации. Вы можете выбрать из следующих программ развертывания:
    - **Microsoft Store для бизнеса**
    - **Microsoft Intune**
    - **Центр партнеров**

## <a name="part-2-set-up-a-windows-10-device-for-microsoft-365"></a>Часть 2: Настройка устройства Windows 10 для Microsoft 365
Прежде чем настраивать устройства Windows для пользователей Microsoft 365, убедитесь, что все устройства Windows работают под управлением Windows 10, версия 1703 (создатели) или более поздней версии.

После обновления всех устройств Windows в Организации до обновления Windows 10 Creators или обновления Windows 10 Creators вы можете присоединить эти устройства к Azure Active Directory вашей организации.

### <a name="set-up-a-brand-new-or-newly-upgraded-windows-10-device"></a>Настройка нового или только что обновленного устройства с Windows 10
Выполните указанные ниже действия, чтобы настроить устройство с помощью ПРИВЕТСТВИя Windows 10 с новым устройством, на котором выполняется обновление Windows 10 Creators (или более поздней версии), или на устройстве, которое было обновлено до Windows 10 Creators (или более поздней версии), но не проходилось с помощью готовой программы установки.

1. Если у вас нет беспроводной сети, убедитесь, что устройство подключено к Интернету через проводное или Ethernet подключение.
2. Пройдите процесс установки устройства Windows. На новом устройстве или перезагрузке с помощью программы установки начинается с региона "Приступая к работе" **. Это правильно?** странице.
3. Начните настройку Windows 10 на устройстве и дойдите до страницы **Выбор способа настройки**. Выберите **Настройка для Организации**.
4. Войдите в систему, используя учетную запись и пароль пользователя Microsoft 365. В зависимости от настройки пароля пользователя может быть предложено обновить пароль. 
5. Завершите настройку Windows 10 на устройстве.

После этого устройство будет подключено к Azure AD вашей организации.

### <a name="set-up-a-device-that-has-already-completed-out-of-box-setup"></a>Настройка устройства, которое уже выполнило установку после завершения работы
Если на устройстве имеется обновление Windows 10 Creators (или более поздней версии) и уже прошло установку, выполните указанные ниже действия.

1. На компьютере пользователя с Windows, работающем под управлением Windows 10, версии 1703 (Creators Update), выберите логотип **Windows** , а затем щелкните значок " **Параметры** ".
2. В окне **Параметры** выберите элемент **Учетные записи**.
3. На странице " **сведения** " выберите **доступ к рабочему или учебному** > **подключению**.
4. В диалоговом окне **Настройка рабочей или учебной учетной записи** в разделе **альтернативные действия**выберите **присоединить это устройство к Azure Active Directory**.
5. На странице **получить вход в** систему введите рабочую или учебную учетную запись и нажмите кнопку **Далее**.
6. На странице **Введите пароль** введите пароль и нажмите кнопку **войти**.
7. Убедитесь, что на странице **ваша организация** указана правильная информация, и выберите команду **присоединиться**.
8. На странице **Готово!** Нажмите кнопку **done (Готово**).

После этого пользователь будет подключен к Azure AD вашей организации.

### <a name="verify-the-device-is-connected-to-azure-ad"></a>Проверка подключения устройства к Azure AD
Выполните следующие действия, чтобы проверить состояние синхронизации устройства с помощью Azure AD, а затем начните использовать свою учетную запись Microsoft 365 на устройстве. 

1. Откройте **Параметры**.
2. На странице " **доступ к работе или учебному доступу** " выберите область **подключено <organization name> ** , чтобы отобразить **сведения о** кнопках и **Отключить**ее.
3. Выберите **сведения** , чтобы получить состояние синхронизации.
4. На странице " **состояние синхронизации** " выберите вариант **Синхронизация** , чтобы получить последние политики управления мобильными устройствами на компьютер.
5. Чтобы начать работу с учетной записью Microsoft 365, перейдите к кнопке **Пуск** Windows, щелкните правой кнопкой мыши текущий рисунок учетной записи, а затем выберите пункт **Переключить** учетную запись.
6. Войдите, используя адрес электронной почты и пароль организации.

Если у вас возникли проблемы при использовании Windows 10 в корпоративной среде, вы можете обратиться к наиболее [распространенным решениям по поддержке корпорации Майкрософт](https://docs.microsoft.com/windows/client-management/windows-10-support-solutions). Эти ресурсы включают статьи базы знаний, обновления и статьи библиотеки.
