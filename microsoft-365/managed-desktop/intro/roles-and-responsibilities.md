Если ваша организация участвуют в Microsoft Desktop управляемых, что делает Microsoft? для вас И какие обязанности вашей организации?

## <a name="microsofts-roles-and-responsibilities"></a>Роли и обязанности корпорации Майкрософт

Ниже приведены некоторые ключевые ролей и обязанностей, которые будут предоставлены вам корпорацией Майкрософт.

Роль или ответственности | Описание
--- | ---
Управление политиками MDM | Корпорация Майкрософт применять политики MDM в соответствии с рекомендациями и рассмотрите запросов для изменения политики. Мы также будет внести изменения в клиент, описанный в [политики устройств](../service-description/device-policies.md).
Развертывание приложений | Корпорация Майкрософт будет развернуть одобренных приложений для устройств Microsoft Desktop управляемого клиента с помощью Intune. Дополнительные сведения см в [приложениях](../get-ready/apps.md). 
Поддержка конечных пользователей | Корпорация Майкрософт будет поддерживает конечных пользователей для устройств, Windows и Office набора продуктов для всех зарегистрированных пользователей. 
Поддержка службы Microsoft Desktop управляемых | Корпорация Майкрософт предоставляет поддержку для клиента ИТ отделы через управляемого рабочего стола операций группы разработчиков Microsoft. Эта группа будет поддержки поиска и устранения неисправностей, запросы на изменение и управления происшествиями для клиента Microsoft Desktop управляемой среды. Дополнительные сведения можно [получить поддержку](../service-description/support.md).
Мониторинг безопасности | Корпорация Майкрософт будет следить за устройств Microsoft Desktop управляемого клиента с помощью ATP Защитника Windows. Если угрозы обнаружено с Microsoft управляемого рабочего стола безопасности операции центр ответственности — Microsoft уведомляет клиента, изолировать устройства и исправления ошибки, в удаленном режиме. [Для получения дополнительных сведений см.](../service-description/security.md)
Обновление мониторинг и управление | Microsoft активное наблюдение за клиента Microsoft Desktop управляемых устройств, убедитесь, что установлены последние обновления качества, компонента и определения для Microsoft Windows и Microsoft Office. Для получения дополнительных сведений см [порядок обработки обновлений](../service-description/updates.md).
Закупка устройства | Microsoft будет размещения доставку упорядоченном устройства Microsoft Desktop управляемых конечным пользователям или точка распространения ИТ по выбору. Для получения дополнительных сведений см. [устройства](../get-started/devices.md).

## <a name="your-roles-and-responsibilities"></a>Роли и обязанности

Ниже приведен дополнительный набор типичные роли, которые не предоставляются корпорацией Майкрософт, но необходимы для успешного развертывания. Не исчерпывающий, но обычно для большинства организаций. 

Роль или reponsibility | Описание
--- | ---
Управление изменениями | Microsoft Desktop управляемых будут уведомлять клиентов, заранее, изменения должны быть выполнены для их Microsoft Desktop управляемой среды. Пользователи должны иметь свои собственные процедуры управления изменениями и необходимые для контактов на рабочий стол управляемых Microsoft. Клиент должен ресурсов для просмотра и утверждения этих изменений. Дополнительные информация в разделе [операции и мониторинг](../service-description/operations-and-monitoring.md).  
Управление удостоверениями | Создание учетных записей пользователей, назначение пользователей в группы и защита метаданных в актуальном состоянии. 
Управление и настройка Office 365 | Администрирование Exchange Oonline, включая:<br>-Администрирование электронной почты<br>-Конфигурация mailbox и правила<br>-Локальную систему Exchange управления<br><br>Средства совместной работы, администрирования SharePoint server, Управление доменами, безопасности и сведения о политиках в портал администрирования Office 365 (Microsoft Desktop управляемых обеспечит приложения Office развертываются на устройства конечных пользователей и своевременно). 
Поддержка конечных пользователей | Клиент будет предоставлять поддержку конечных пользователей: <br>-На веб-узла: все сети и подключение к Интернету, VPN инфраструктуры и клиента конфигурации, локального конференции комнаты оборудования, принтеры, прокси-сервера и конфигурации, брандмауэры.<br><br>-Ресурсы cloud компании в целом: электронной почты, SharePoint, служб для совместной работы и другие облачной инфраструктуры, связанные с газа технологии всей компании.<br><br>-Из бизнес-приложений: пользовательского программного обеспечения, программного обеспечения сторонних производителей, программное обеспечение planning (ERP) корпоративных ресурсов, средства разработки и ничего не указано в настоящем документе.
Пользователь и группировки устройства | Группы разработчиков Microsoft Desktop управляемых операции будет создание и управление ими необходимые устройства и групп пользователей в составе ИТ-операций. Клиент не будет внести изменения в эти группы без первого общение ИТ-операций по каналам поддерживаемые. Все обнаруженные изменения будут отменены.
Приложения | Клиенты предоставит список приложений, которые они хотят использовать в своей организации (превышающее приложения, включенные в состав Microsoft 365 лицензии). Клиенты также предоставляют пакет развертывания (Appx или MSI)<br>Несет ответственность за клиента:<br>-Управление лицензиями app – с правом тип и количество лицензий<br>-Назначение app – назначение приложений для групп пользователей Azure AD<br>-Обновления app – мониторинг упаковки и развертывания приложения функции и обновления безопасности<br>-Приложение пользователей тестирования (UAT)<br><br>Дополнительные сведения см в [приложениях](../get-ready/apps.md)
Мониторинг безопасности и ответа | При обнаружении происшествие безопасности, которой выходит за рамки Microsoft Operations управляемого рабочего стола, мы потребует списка контактов привилегированного клиента на основании серьезность проблемы.<br><br>Клиент несет ответственность за исследование и разрешение происшествия для устройств не Microsoft управляемых ПК и проверка того, что группа операций рабочего стола управляемых корпорации Майкрософт получает сведения о проблемы, которые могут влиять на службу.
Поддержка операций | Microsoft Operations управляемого рабочего стола необходим список контактов привилегированного клиента и экспертов. Мы необходимые на случай действующие происшествия не Microsoft управляемых ПК. <br><br>Клиент несет ответственность за исследование и разрешение происшествия для устройств не Microsoft управляемых ПК и служб и проверка того, что группа операций рабочего стола управляемых корпорации Майкрософт всегда уведомляется.
Сетевая инфраструктура, включая VPN | Программа установки, настройки и управления (включая устранение неполадок и отладка) все связанные с сетью инфраструктуры и службы, включая подключение к Интернету, элементы управления, настройки прокси-сервера и сети инфраструктуры удаленного подключения.<br><br>По умолчанию Microsoft Desktop управляемых укажите или не требуют использования прокси-сервера. Тем не менее если она настроена (в оборудования или программного обеспечения), есть коллекция URL-адресов, которые должны быть включены прокси-сервером. Клиент несет ответственность за устранение неполадок все конфликты или несовместимости, из-за несколько прокси-серверов.<br><br>Для получения дополнительных сведений см [С прокси-сервером](../get-ready/network.md).
Печать | Установка, обслуживание и администрирование принтеры и очереди печати. Облако печати — это рекомендуемое решение, но не является обязательным. 
Мобильные устройства | Устройства и стандартные, не предоставленные управляемых Microsoft Desktop. Управляемые группы операций рабочего стола Microsoft поддерживает только устройства Microsoft Desktop управляемых, стыковочного узла и включены стандартные.
