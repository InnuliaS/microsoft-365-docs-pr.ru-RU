Если ваша организация участвуют в Microsoft Desktop управляемых, что делает Microsoft? для вас И какие обязанности вашей организации?

## <a name="microsofts-roles-and-responsibilities"></a>Роли и обязанности корпорации Майкрософт

Ниже приведены некоторые ключевые ролей и обязанностей, которые будут предоставлены вам корпорацией Майкрософт.

Роль или ответственности | Описание
--- | ---
Управление политиками MDM | Корпорация Майкрософт применять политики MDM в соответствии с рекомендациями и рассмотрите запросов для изменения политики. Мы также будет внести изменения в клиент, описанный в [политики устройств](../service-description/device-policies.md).
Поддержка конечных пользователей | Корпорация Майкрософт будет поддерживает конечных пользователей для устройств, Windows и Office набора продуктов для всех зарегистрированных пользователей по приложению получить помощь, предварительно установленное на все устройства Microsoft управляемых Desktop. 
Поддержка службы Microsoft Desktop управляемых | Корпорация Майкрософт предоставляет поддержку для клиента ИТ отделы через управляемого рабочего стола операций группы разработчиков Microsoft. Эта группа будет поддержки поиска и устранения неисправностей, запросы на изменение и управления происшествиями для клиента Microsoft Desktop управляемой среды. Для получения дополнительных сведений см [Поддержка Microsoft Desktop управляемых администрирования](../working-with-managed-desktop/admin-support.md).
Мониторинг безопасности | Корпорация Майкрософт будет следить за устройств Microsoft Desktop управляемого клиента с помощью ATP Защитника Windows. Если угрозы обнаружено с Microsoft управляемого рабочего стола безопасности операции центр ответственности — Microsoft уведомляет клиента, изолировать устройства и исправления ошибки, в удаленном режиме. [Для получения дополнительных сведений см.](../service-description/security.md)
Обновление мониторинг и управление | Microsoft активное наблюдение за устройства Microsoft Desktop управляемого клиента для проверки установки последних обновлений качество и компонентов для Microsoft Windows и Microsoft Office. Для получения дополнительных сведений см [порядок обработки обновлений](../service-description/updates.md).
Пользователь и группировки устройства | Группы разработчиков Microsoft Desktop управляемых операции будет создание и управление ими необходимые устройства и групп пользователей в составе ИТ-операций. Не должна быть изменения, внесенные в эти группы без утверждения из группы разработчиков Microsoft Desktop управляемых операций.

## <a name="your-roles-and-responsibilities"></a>Роли и обязанности

Ниже приведен дополнительный набор распространенных ролей и обязанностей, которые не предоставляются корпорацией Майкрософт, но необходимы для успешного развертывания. Он не исчерпывающий, но можно применять для большинства организаций. Существует несколько элементов под Microsoft и клиента общий доступ к responsibilties. 

Роль или ответственности | Описание
--- | ---
Управление изменениями | Microsoft будут уведомлять клиентов, заранее, изменения должны быть выполнены для их Microsoft Desktop управляемой среды. Клиент, должен иметь свои собственные управление изменение обработки и контакт с группы разработчиков Microsoft Operations управляемого рабочего стола. Клиент также требуется для ресурсов для просмотра и утверждения этих изменений. Для получения дополнительных сведений см [операции и мониторинг](../service-description/operations-and-monitoring.md).  
Управление удостоверениями | Клиент несет ответственность за создание учетных записей пользователей, назначение пользователей в группы и сохранение метаданных в актуальном состоянии. 
Управление и настройка Office 365 | Microsoft несет ответственность за обеспечение развертывания приложений Office для конечных пользователей и этих приложений хранятся в актуальном состоянии. <br><br> Клиент несет ответственность за управление служб Office 365 и политики, включая Exchange Online обязанности администрирования:<br>-Администрирование электронной почты<br>-Конфигурация mailbox и правила<br>-Локальную систему Exchange управления<br><br>Клиент также отвечает за средства совместной работы, администрирования SharePoint server, Управление доменами, политики безопасности и данные в портал администрирования Office 365. 
Поддержка конечных пользователей | Клиент отвечает за предоставление поддержки конечным пользователем для: <br>-На веб-узла: все сети и подключение к Интернету, VPN инфраструктуры и клиента конфигурации, локального конференции комнаты оборудования, принтеры, прокси-сервера и конфигурации, брандмауэры.<br><br>-Ресурсы cloud компании в целом: электронной почты, SharePoint, служб для совместной работы и другие облачной инфраструктуры, связанные с газа технологии всей компании.<br><br>-Строка бизнеса и любой другой компании определенные приложения.
Приложения | Корпорация Майкрософт предоставляет руководство по развертыванию для одобренных приложений, использующих Intune по запросу.<br><br>Несет ответственность за клиента:<br>-Предоставление списка приложений для своей организации (вне приложения Office 365)<br>-Управление лицензиями app<br>– С правом тип и количество лицензий<br>-Назначение app<br>– Назначение приложений для групп пользователей Azure AD<br>-Обновляет app<br>– Мониторинг, упаковки и развертывания приложения функции и обновления безопасности<br>-Приложение пользователей тестирования (UAT)<br>-Предоставление подходящего пакет развертывания<br><br>Дополнительные сведения см в [приложениях](../get-ready/apps.md)
Мониторинг безопасности и ответа | Клиент несет ответственность за исследование и разрешение происшествия для устройств не Microsoft управляемых ПК и проверка того, что группа операций рабочего стола управляемых корпорации Майкрософт получает сведения о проблемы, которые могут влиять на службу.
Поддержка операций | Клиент отвечает за предоставление списка контактов привилегированного клиента и экспертов. Microsoft должно их на случай действующие происшествия не Microsoft управляемых ПК. <br><br>Клиент также несет ответственность за исследование и разрешение происшествия для устройств не Microsoft управляемых ПК и служб и проверка того, что группа операций рабочего стола управляемых корпорации Майкрософт всегда уведомляется.
Сетевая инфраструктура, включая VPN | Клиент несет ответственность за программы установки, настройки и управления (включая устранение неполадок и отладка) все инфраструктуры, связанные с сетью и служб, включая подключение к Интернету, элементы управления, настройки прокси-сервера и сети удаленного Инфраструктура подключения.<br><br>Если прокси-сервер настроен (в оборудования или программного обеспечения), есть коллекция URL-адресов, которые должны быть включены прокси-сервером. Клиент несет ответственность за устранение неполадок все конфликты или несовместимости, из-за несколько прокси-серверов.<br><br>Для получения дополнительных сведений см [С прокси-сервером](../get-ready/network.md).
Печать | Клиент несет ответственность за установку, обслуживание и администрирование принтеры и очереди печати. Облако печати — это рекомендуемое решение, но не является обязательным.
