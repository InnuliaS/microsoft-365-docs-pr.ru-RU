## <a name="change-management"></a>Управление изменениями

В службе предложение, баланс ответственность за обслуживание оборудования и безопасности обновлений сдвигает к поставщику услуг (Microsoft) вместо клиента (вы). Тем не менее все еще необходимо убедиться, что сторонних производителей и пользовательского программного обеспечения по-прежнему производится так, как ожидалось после развертывания обновлений.

Для продуктов в локальной организации предполагается, что все ответственность за управление изменениями.

## <a name="device-wipe-with-factory-reset"></a>Очистки устройства с фабрики reset

Managed группы операций рабочего стола можно сделать фабрика Сброс на управляемых Microsoft Desktop управляемых устройств, которые должны быть повторного создания образа. Это полезно, если вам нужно передать другому сотруднику устройства или сотрудник покидает вашей компании. 

Существует несколько требований:

- Администратор клиента клиента необходимо отправить запрос на обслуживание
- Нам нужно имя компьютера для устройства
- Учетная запись пользователя должна быть в Azure AD сначала Сброс

Управляемые группы операций рабочего стола выполняет следующие действия:

- Выберите нужное имя устройства в Intune
- Отправить по сбросу фабрики команды на устройство

>[!NOTE]
>Не удаляйте из Azure AD перед фабрики Сброс учетной записи пользователя. Если пользователь не находится в Azure AD, Intune не удается отправить фабрики Сброс команды на устройство. 

Устройство будет загружен в окно Приветствия и все предварительно установленные приложения и параметры будут применены еще раз. Устройство пользователя необходимо предоставить начальный набор сведений о еще раз. 

Когда устройства был изменен, можно передать его другому пользователю в вашей организации. Нет данных или корпоративных данных предыдущего пользователя будет на устройстве. Далее пользователь проходит через тот же самый процесс, который выполнил предыдущего пользователя с помощью нового устройства Microsoft управляемых Desktop.

BitLocker является основным компонентом защиты данных в этом процессе. С помощью шифрования BitLocker на устройствах Microsoft Desktop управляемых данных на диске, остается безопасной даже после сброса фабрики применен на устройство. Все данные на диске, будет недоступен для следующего пользователя устройства.
