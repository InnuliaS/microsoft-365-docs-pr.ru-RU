## <a name="data-security"></a>Безопасность данных

Данные, передаваемые от клиента хранятся в базах данных Azure SQL в клиентов корпорации Майкрософт, размещенные в США. Данные хранятся и соответствует требованиям для размещения Azure безопасности. 

Для получения дополнительных сведений см [Microsoft Azure](https://www.microsoft.com/TrustCenter/Security/azure-security).

В этом списке перечислены типы данных, передаваемых между корпорацией Майкрософт и вашего клиента. 

- От корпорации Майкрософт в клиент
    - Имена групп
    - Настройка политики безопасности
    - Устройство заказы
    - Учетные записи пользователей (msadmin, mstest Desktop_soc_ro управляемых Microsoft, Desktop_wdgsoc управляемых Microsoft)
    - Назначение лицензий E5 выше 4 пользователям
    - Политики для звонков обновления Центра обновления Windows
    - В будущем дополнительные возможности, которые будут реализованы позволяет публиковать сведения о других типов содержимого конфигурации, включая приложения, политик и параметров.
- От клиента в корпорацию Майкрософт
    - Устройство обновления, об использовании и надежности данных
    - Развертывание и надежности данных приложения
    - Данные развертывания политики безопасности и обновления
    - Пользователи, которым назначены устройств



## <a name="device-security"></a>Безопасность устройств

Для предотвращения нарушения системы безопасности, важно убедитесь, что все устройства Microsoft Desktop управляемых с защитой и работоспособность. Не менее важно обеспечить мы определить неработоспособные устройства и снизить риск как можно раньше.

В следующей таблице приведены служб, предоставляемых убедитесь, что устройства Microsoft Desktop управляемых надежных, работоспособны и с защитой.

Служба | Описание
--- | ---
Защита от вирусов | Мы будет убедитесь, что:<br>-Устанавливается и настраивается Защитник Windows аудио/видео<br>-Определения Защитника Windows аудио/видео в актуальном состоянии
Полное шифрование тома |    Windows BitLocker — решение шифрования данных для Microsoft Desktop управляемых устройств.<br><br>После организации onboarded в службу, устройств шифруются с помощью Windows BitLocker с помощью встроенных доверять платформы модуля (TPM) для предотвращения несанкционированного доступа для локальных данных, если устройство находится в спящем режиме или отключено. 
Отслеживание |    Windows Защитник расширенного защиту от угроз (ATP Защитника Windows) — это угроз безопасности, наблюдения за решением для всех устройств Microsoft Desktop управляемых. Дополнительные сведения можно [защиту от угроз дополнительные Защитник Windows.](https://docs.microsoft.com/windows/threat-protection/windows-defender-atp/windows-defender-advanced-threat-protection) 
Обновления программного обеспечения |  Microsoft убедитесь, что устройства Microsoft Desktop управляемых всегда защищено последние обновления для Windows и Office, с помощью центра обновления Windows для бизнеса.
Восстановление |  Корпоративные данные хранятся на OneDrive для бизнеса и быстро восстановить для Microsoft Desktop управляемых устройств. Дополнительные сведения можно [OneDrive для бизнеса.](https://support.office.com/article/Restore-a-previous-version-of-a-file-in-OneDrive-159cad6d-d76e-4981-88ef-de6e96c93893?ui=en-US&rs=en-US&ad=US) 



## <a name="identity-security"></a>Идентификатор безопасности

Управление удостоверениями и доступом обеспечивает защиту корпоративных активов и важными бизнес-данных. Azure Active Directory (Azure AD) предоставляет службы работы с удостоверениями в облаке и enable облачной проверки подлинности, который обеспечивает только надежные пользователи доступ к корпоративным ресурсам с помощью Microsoft Desktop управляемых устройств.

Служба | Описание
--- | ---
Поставщик проверки подлинности марки предприятия |  Azure AD расширенных выпусков, используемые корпорацией Майкрософт предоставление высокого уровня доступности службы, размещенной в глобальное распределение центров обработки данных. Служба обрабатывает миллиардов проверок подлинности каждого дня, с более чем 200 миллионов активных пользователей и дает SLA 99,9%.
Биометрическая проверка подлинности |  Windows Hello пользователи могут войти в систему с помощью своих фрагмент или ПИН-код, что пароли труднее забыли или похитить. Может потребоваться настройка дополнительных работ. Дополнительные сведения можно [Windows Hello.](https://docs.microsoft.com/windows-hardware/design/device-experiences/windows-hello)
Настройка проверки подлинности | Многофакторная проверка подлинности Azure не позволяет несанкционированного доступа для локальных и облачных приложений, предоставляя дополнительный уровень проверки подлинности с помощью мобильного телефона, а также как самостоятельного сброса пароля. 
Разрешение обычного пользователя |  Чтобы защитить систему и сделать его более безопасной, пользователь будет назначен стандартные разрешения пользователя. Назначена в качестве стандартной автопилот Windows при работе.

## <a name="privileged-account-access"></a>Привилегированной учетной записи доступа

В настоящее время мы ограничить доступ к MSAdmin учетные данные клиента и приложения через следующие механизмы:

- **Операторы** – Microsoft full-раз-сотрудников в США, которые успешно завершена периодическую проверку безопасности и фона.
- **Оператор удостоверения** — для защиты стандарты, заданным корпорацией Майкрософт. Дополнительные сведения содержатся в разделе [Управление учетными данными пользователей и безопасный доступ в корпорации Майкрософт](https://www.microsoft.com/itshowcase/Article/Content/931/Managing-user-identities-and-secure-access-at-Microsoft). 
- **Сбор данных** выполняется в среде оператор и в клиент пользователя. Данные журнала и персональные данные сохраняются в соответствующих средах и не проходят через сред.
