# <a name="device-monitoring-and-reporting-in-the-microsoft-365-security-center"></a>Мониторинг устройств и создание отчетов в центре безопасности Майкрософт 365

Обеспечьте безопасность и актуальность ваших устройств, а также возможность выявления потенциальных угроз в центре безопасности Microsoft 365.

## <a name="view-device-alerts"></a>Просмотр оповещений устройства

Получение обновленных предупреждений о действиях с нарушениями появления и других угрозах на устройствах из пакета ATP для Microsoft Defender (доступно с помощью лицензии "по"). Центр безопасности Microsoft 365 эффективно отслеживает эти оповещения на высоком уровне с помощью предпочтительного рабочего процесса.

### <a name="monitor-high-impact-alerts"></a>Отслеживание оповещений о высокой степени влияния

Каждое оповещение для защитника Microsoft Defender имеет соответствующее серьезность, среднюю, низкую или информационную, которая указывает на потенциальное влияние на сеть, если оно не указано в автоматическом режиме.  

Используйте карточку **серьезности оповещений для устройств** , чтобы сосредоточиться на более серьезных оповещениях и может требовать немедленный отклик. С помощью этой карточки вы можете просмотреть дополнительные сведения на портале Центра безопасности защитника Майкрософт.

![Карточка серьезности оповещений устройств](../images/device-alerts-severity.png)

### <a name="understand-sources-of-alerts"></a>Общие сведения об источниках оповещений

Защитник Майкрософт использует данные из широкого спектра датчиков безопасности и источников аналитики для создания оповещений. Например, он может использовать сведения об обнаружении антивирусной программы "Защитник Windows" и стороннего антивредоносного по, а также собственную логику, предоставляемую через API веб-службы.

В карточке источников **обнаружения оповещений для устройств** показано распределение оповещений по источнику. С помощью этой карточки можно отслеживать действия, связанные с определенными источниками, в частности настраиваемые источники. Кроме того, вы можете сосредоточиться на оповещениях, поступающих от датчиков, которые не настроены для автоматической блокировки вредоносных действий или компонентов.

![Карточка источников обнаружения оповещений для устройств](../images/device-alert-detection-sources.png)

С помощью этой карточки вы можете просмотреть дополнительные сведения на портале Центра безопасности защитника Майкрософт.

### <a name="understand-the-types-of-threats-that-trigger-alerts"></a>Сведения о типах угроз, инициирующих оповещения

Защитник Майкрософт сортирует каждое оповещение в категории, представляющей определенный этап в цепочке атак, или тип компонента угрозы. Например, обнаруженные действия по угрозе могут быть отнесены к Латерал перемещению, чтобы показать, что действие приводило к попытке получить доступ к другим устройствам в сети и, вероятно, произошло после того, как злоумышленник получит первоначальный фусолд. При обнаружении компонент для работы с угрозой может быть классифицирован в широком смысле как вредоносное или более конкретное, например, Кража учетных данных или другие типы вредоносных или нежелательных программ.

В карточке **категории угроз для устройств** показано распределение оповещений по этим категориям. Эти сведения можно использовать для определения действий по угрозам, например попыток кражи учетных данных, которые могут значительно повлиять на появление попыток социального проектирования, например. Кроме того, вы можете использовать эту возможность для отслеживания потенциально необратимых угроз, таких как шантажистом.

![Карточка категорий угроз для устройств](../images/device-threat-categories.png)

### <a name="monitor-active-alerts"></a>Отслеживание активных оповещений

Карточка **состояния оповещения для устройства** указывает количество оповещений, которые не были разрешены и могут потребовать внимания. С помощью этой карточки вы можете просмотреть дополнительные сведения на портале Центра безопасности защитника Майкрософт.

![Карточка состояния оповещения для устройства](../images/device-alert-status.png)

### <a name="monitor-classification-of-resolved-alerts"></a>Отслеживание классификации разрешенных оповещений

При разрешении оповещения Microsoft Defender ATP сотрудники могут указать, было ли оповещение проверено следующим образом:

* Истинное оповещение, идентифицирующее реальные действия или компоненты угроз
* Ложное оповещение, которое неправильно определило нормальные действия

На карточке **классификация оповещений устройств** показано, были ли разрешенные оповещения классифицированы как "истина" или "ложные". С помощью этой карточки вы можете просмотреть дополнительные сведения на портале Центра безопасности защитника Майкрософт.

Примечание. в некоторых случаях сведения об классификации недоступны для определенных оповещений.

![Карточка классификации оповещений для устройств](../images/device-alert-classification.png)

### <a name="monitor-determination-of-resolved-alerts"></a>Отслеживание определения разрешенных оповещений

В дополнение к классификации того, является ли оповещение истинным или ложным во время разрешения, персонал по обеспечению безопасности может определить тип обычной или вредоносной активности, обнаруженной при проверке оповещения.

На карточке **определения оповещения для устройства** отображается определение, предоставленное для каждого оповещения, а именно:

* **Апт** Advanced persistent Threat угроза, указывающая на то, что обнаруженное действие или компонент угроз является частью сложного нарушения безопасности, разработанного для получения фусолд в сети.  
* Вредоносный файл вредоносных **программ** или код
* **Нормальная работа персонала безопасности,** выполняемая сотрудниками по безопасности
* Действия по **тестированию системы безопасности** или компоненты, предназначенные для моделирования фактических угроз и ожидаемые для запуска датчиков безопасности и создания оповещений
* **Нежелательные программные** приложения и другое программное обеспечение, которые не являются вредоносными, но в противном случае нарушают политику или допустимые стандарты использования
* **Другие определения** , не попадающие в указанные типы

Из этой карточки вы можете просмотреть дополнительные сведения в центре безопасности защитника Майкрософт.

![Карточка определения оповещения для устройства](../images/device-alert-determination.png)

### <a name="understand-which-devices-are-at-risk"></a>Сведения о том, какие устройства подвержены риску

**Защита устройства** показывает уровень риска для устройств. Уровень риска зависит от таких факторов, как тип и серьезность оповещений на устройстве.

## <a name="monitor-and-report-status-of-intune-managed-devices"></a>Отслеживание и отправка отчетов о состоянии устройств, управляемых Intune

Следующие отчеты содержат данные от устройств, зарегистрированных в Intune. Данные из незарегистрированных устройств не входят в состав. Просматривать эти карты могут только глобальные администраторы.

Данные устройства, зарегистрированные в Intune, включают:

* Соответствие устройства требованиям
* Устройства с активным вредоносным по
* Типы вредоносных программ на устройствах
* Вредоносные программы на устройствах
* Устройства с обнаружением вредоносных программ
* Пользователи с обнаружением вредоносных программ

## <a name="monitor-and-manage-asr-rule-deployment-and-detections"></a>Мониторинг развертывания и обнаружения правил ASR и управление ими

[Правила предотвращения снижения уязвимости (ASR)](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/attack-surface-reduction) помогают предотвратить действия и приложения, которые обычно используются вредоносной программой для проникновения устройств. Эти правила определяют, когда и как запускаются исполняемые файлы. Например, вы можете предотвратить запуск загруженного исполняемого файла в JavaScript или VBScript, блокировать вызовы Win32 API из макросов Office или блокировать процессы, которые выполняются с USB-накопителей.

![Карточка о снижении уязвимости](../images/attack-surface-reduction-rules.png)

На карточке **Правила сокращения направлений атак** приведены общие сведения о развертывании правил на устройствах.

В верхнем столбце на этой карточке показано общее количество устройств, которые находятся в следующих режимах развертывания:

* **Блочные** устройства с как минимум одним правилом, настроенными для блокирования обнаруженных действий
* Устройства **режима аудита** без набора правил, блокирующие обнаруженные действия, но имеющие по крайней мере один набор правил для аудита обнаруженных действий  
* **Отключение** устройств, для которых отключены все правила ASR

В нижней части этой карточки приведены параметры всех устройств, отсортированные по правилам. В каждом столбце указано количество устройств, для которых либо настроена блокировка или аудит обнаруженных действий, либо полностью отключено соответствующее правило.

### <a name="view-asr-detections"></a>Просмотр обнаруженных аварийных ASR

Чтобы просмотреть подробные сведения об обнаружении правил ASR в сети, выберите **Просмотр обнаружений** в карточке **правила уменьшения уязвимой зоны** . Откроется вкладка **обнаружения** на странице подробный отчет.

![Вкладка "обнаружения"](../images/detections-tab.png)

На диаграмме, расположенной в верхней части страницы, показано обнаружение с обнаружением стеков по времени, которые были заблокированы или проверены. В таблице внизу перечислены последние обнаруженные действия. Чтобы понять, в чем заключаются обнаруженные действия, используйте следующие сведения в таблице:

* **Обнаруженный файл** файл (как правило, сценарий или документ), содержимое которого активировало вероятные действия по атакам
* Имя **правила** , описывающее действия, которые разрабатываются правилом для перехвата. Сведения о существующих правилах ASR
* **Исходное приложение** приложение, которое загрузило или выполнило содержимое, вызывающее вероятные действия по атакам. Это может быть легальное приложение, например веб-браузер, приложение Office или системное средство, например PowerShell.
* **Publisher** — поставщик, который освободил исходное приложение

### <a name="review-device-asr-rule-settings"></a>Проверка параметров правил для устройства ASR

На странице отчета " **правила снижения уязвимости** " перейдите на вкладку **Конфигурация** , чтобы просмотреть параметры правил для отдельных устройств. Выберите устройство, чтобы получить подробные сведения о том, находится ли каждое правило в режиме блокировки, режиме аудита или полном отключении.

![Вкладка "Конфигурация"](../images/configuration-tab.png)

Microsoft Intune предоставляет функции управления для правил ASR. Если вы хотите обновить параметры, выберите **начать работу** в разделе **Настройка устройств** на вкладке, чтобы открыть компонент "Управление устройствами в Intune".

### <a name="exclude-files-from-asr-rules"></a>Исключение файлов из правил ASR

Центр безопасности Microsoft 365 собирает имена [файлов, которые могут потребоваться исключить](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/enable-attack-surface-reduction#exclude-files-and-folders-from-asr-rules) из обнаруженных правил по снижению уязвимости. За счет исключения файлов можно сократить количество ложных срабатываний и более уверенно развернуть правила уменьшения уязвимой зоны в режиме блокировки.

Исключения управляются в Microsoft Intune, но центр обеспечения безопасности Microsoft 365 предоставляет средство анализа, помогающее ознакомиться с файлами. Чтобы начать сбор файлов для исключения, перейдите на вкладку " **Добавление исключений** " на странице отчета " **правила уменьшения уязвимой зоны** ".

>[!NOTE]  
>Средство анализирует обнаружение по всем правилам сокращения направлений атак, но [только некоторые правила поддерживают исключения](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/troubleshoot-asr).

![Вкладка "Добавление исключений"](../images/add-exclusions-tab.png)

В таблице перечислены все имена файлов, обнаруженные правилами сокращения для атак на уязвимую зону. Вы можете выбрать файлы, чтобы проанализировать последствия их исключения:

* Сколько меньше обнаруженных обнаружений
* Сколько устройств сообщает об обнаружении

Чтобы получить список выбранных файлов с полными путями для исключения, выберите пункт **Получение путей исключения**.

Журналы для **переноса учетных данных блока правил ASR из подсистемы Windows Local Security Authority (LSASS. exe)** захватывает исходное приложение **LSASS. exe**, обычный системный файл, как обнаруженный файл. В результате созданный список исключаемых путей будет содержать этот файл. Чтобы исключить файл, который инициировал это правило, а не **LSASS. exe**, используйте путь к исходному приложению, а не к обнаруженному файлу.

Чтобы найти исходное приложение, выполните следующий [Расширенный запрос поиска](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-hunting) для этого конкретного правила (определяемого с помощью идентификатора правила 9e6c4e1f-7d60-472f-ba1a-a39ef669e4b2):

```kusto
MiscEvents
| where EventTime > ago(7d)
| where ActionType startswith "Asr"
| where AdditionalFields contains "9e6c4e1f-7d60-472f-ba1a-a39ef669e4b2"
| project InitiatingProcessFolderPath, InitiatingProcessFileName
```

#### <a name="check-files-for-exclusion"></a>Проверка файлов для исключения
Перед исключением файла из ASR рекомендуется проверить его, чтобы определить, не является ли он вредоносным.

Чтобы просмотреть файл, используйте [страницу сведений о файле](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/investigate-files) в центре безопасности защитника Майкрософт. На странице предоставляются сведения о распространении, а также степень обнаружения антивирусной программы Вирустотал. Вы также можете использовать эту страницу для передачи файла для детального анализа.

Чтобы найти обнаруженный файл в центре безопасности защитника Майкрософт, выполните поиск всех обнаружений ASR, используя следующий Расширенный запрос поиска:

```kusto
MiscEvents
| where EventTime > ago(7d)
| where ActionType startswith "Asr"
| project FolderPath, FileName, SHA1, InitiatingProcessFolderPath, InitiatingProcessFileName, InitiatingProcessSHA1
```

Используйте **SHA1** или **InitiatingProcessSHA1** в результатах поиска файла с помощью универсальной панели поиска в центре безопасности защитника Microsoft.
